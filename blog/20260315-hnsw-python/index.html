向量搜索的核心算法：用 Python 从零实现 HNSW

⏱ 2026年3月15日
· 33 分钟阅读
#python#ai#vector-search#hnsw#algorithm

最近在搭一个 RAG 系统，用到了向量数据库做语义检索。Pinecone、Qdrant、Weaviate、Milvus……选型文章到处都是，但翻来翻去我发现大家都在讨论「用哪个」，很少有人关心「它底层怎么搜的」。

向量搜索这个事情，外面包了一层 SDK 之后就变成了黑箱。你往里面塞 embedding，调一下 search(query, top_k=10)，结果就回来了。但它到底怎么在百万量级的高维向量里快速找到最相似的那几个？

背后几乎都是同一个算法：HNSW（Hierarchical Navigable Small World）。

我花了两天时间用纯 Python 把它从零实现了一遍。

暴力搜索为什么撑不住

最直觉的做法当然是暴力搜索——遍历所有向量，逐一算距离，取最小的 k 个。复杂度 O(n)，1000 条向量的时候完全没问题，numpy 一把梭，零点几毫秒就出结果。

但到了 100 万条、1000 万条的规模呢？每次查询都要算 100 万次距离，延迟直接上天。线上服务根本没法接受这种响应时间。

所以我们需要近似最近邻（ANN）算法——用一点点精度换巨大的速度提升。HNSW 是目前效果最好的 ANN 算法之一，在高召回率和低查询延迟之间取得了很好的平衡。

HNSW 的核心思路

HNSW 的灵感来源可以类比跳表（skip list），但它操作的对象不是有序链表，而是图。

想象你要在一个城市里找最近的咖啡店。暴力搜索就是挨家挨户问过去。HNSW 的做法是建了一套分层地图：

  • 顶层：只标了几个主要地标，节点少、边长，像高速公路网络，一步能跨很远
  • 中间层：节点多一些，粒度更细
  • 底层（layer 0）：所有节点都在，密密麻麻，像城市街道

每个节点被随机分配到某一层（指数递减分布——绝大多数节点只在 layer 0，少量在 layer 1，极少数在更高层）。

搜索过程：从最顶层的入口点出发，贪心地找当前层里离目标最近的节点，找不到更近的了就下降一层，继续。到了 layer 0 之后，展开一个更宽的搜索（beam search），用 ef 参数控制搜索宽度。

这个分层结构让搜索复杂度从 O(n) 降到了 O(log n)。

关键实现细节

几个不看论文很容易搞错的地方：

随机层数的生成：不是均匀随机，是几何分布。ml = 1/ln(M)，然后 level = floor(-ln(uniform_random) * ml)。M 是每个节点的最大连接数。这个分布保证了高层节点数量指数递减。

layer 0 更密：底层连接数上限 M_max0 = 2*M，比其他层多一倍。这是因为 layer 0 承载了最终的精细搜索，连接越密，召回率越高。

邻居选择的启发式：论文里的 Algorithm 4。不是简单取最近的 M 个邻居，而是倾向于选择方向多样的邻居。具体来说：对候选邻居排序，依次考察每个候选——如果它比已选中的任何邻居都更远，就跳过它。这样选出来的邻居分布更均匀，搜索效率更高。

双向边 + 剪枝：插入节点 A 时连接了节点 B，那 B 也要反向连 A。如果 B 的连接数超限了，就要剪掉一些边。

一个让我调了半天的 bug

剪枝逻辑里我一开始只处理了正向：当节点 B 的连接数超过 M_max 时，用启发式选出新的邻居集合，然后直接替换 graph[B]。

结果 recall 一直上不去，怎么调参都没用。

排查了很久才发现问题：假设剪枝时 B 跟节点 C 断开了，graph[B] 里已经没有 C 了，但 graph[C] 里还保留着 B。图的双向性被破坏了。搜索的时候，从 C 出发能到 B，但从 B 出发到不了 C，路径变成了单行道。

修法是：剪掉 B→C 的时候，同时把 C→B 也删掉。

for old_n in neighbor_conns - new_set:
    self.graphs[layer][old_n].discard(neighbor_id)
self.graphs[layer][neighbor_id] = new_set

就这两行的顺序和逻辑，花了我大半个下午。

完整实现

下面是完整的 hnsw.py，包括 L2 距离和余弦距离两种度量：

"""
HNSW (Hierarchical Navigable Small World) implementation for approximate nearest neighbor search.

Implements the algorithm described in:
    Malkov, Y.A., Yashunin, D.A. (2018). Efficient and robust approximate nearest neighbor search
    using Hierarchical Navigable Small World graphs.
"""

import math
import random
from typing import Callable, List, Optional, Tuple

import numpy as np


def l2_distance(a: np.ndarray, b: np.ndarray) -> float:
    diff = a - b
    return float(np.sqrt(np.dot(diff, diff)))


def cosine_distance(a: np.ndarray, b: np.ndarray) -> float:
    dot = np.dot(a, b)
    norm_a = np.linalg.norm(a)
    norm_b = np.linalg.norm(b)
    if norm_a == 0.0 or norm_b == 0.0:
        return 1.0
    return 1.0 - float(dot / (norm_a * norm_b))


DISTANCE_FUNCTIONS = {
    "l2": l2_distance,
    "cosine": cosine_distance,
}


class HNSWIndex:
    def __init__(
        self,
        dim: int,
        M: int = 16,
        ef_construction: int = 200,
        ef_search: int = 50,
        metric: str = "l2",
        seed: Optional[int] = None,
    ):
        self.dim = dim
        self.M = M
        self.M_max0 = 2 * M
        self.ef_construction = ef_construction
        self.ef_search = ef_search
        self.ml = 1.0 / math.log(M) if M > 1 else 1.0

        if metric not in DISTANCE_FUNCTIONS:
            raise ValueError(f"Unknown metric '{metric}'. Choose from: {list(DISTANCE_FUNCTIONS.keys())}")
        self.metric = metric
        self.dist_func: Callable = DISTANCE_FUNCTIONS[metric]

        self.rng = random.Random(seed)
        self._np_rng = np.random.RandomState(seed)

        self.vectors: List[np.ndarray] = []
        self.graphs: List[dict] = []
        self.entry_point: Optional[int] = None
        self.max_layer: int = -1
        self.node_layers: List[int] = []

    @property
    def num_elements(self) -> int:
        return len(self.vectors)

    def _random_level(self) -> int:
        r = self.rng.random()
        if r == 0.0:
            r = 1e-10
        return int(-math.log(r) * self.ml)

    def _distance(self, id_a: int, vec_b: np.ndarray) -> float:
        return self.dist_func(self.vectors[id_a], vec_b)

    def _search_layer(
        self, query: np.ndarray, entry_points: List[int], ef: int, layer: int
    ) -> List[Tuple[float, int]]:
        graph = self.graphs[layer]
        visited = set(entry_points)

        candidates = []
        result = []

        for ep in entry_points:
            d = self._distance(ep, query)
            candidates.append((d, ep))
            result.append((d, ep))

        candidates.sort()
        result.sort()

        while candidates:
            c_dist, c_id = candidates[0]
            f_dist = result[-1][0] if result else float("inf")

            if c_dist > f_dist and len(result) >= ef:
                break

            candidates.pop(0)

            neighbors = graph.get(c_id, set())
            for neighbor in neighbors:
                if neighbor in visited:
                    continue
                visited.add(neighbor)

                d = self._distance(neighbor, query)
                f_dist = result[-1][0] if result else float("inf")

                if d < f_dist or len(result) < ef:
                    candidates.append((d, neighbor))
                    candidates.sort()
                    result.append((d, neighbor))
                    result.sort()
                    if len(result) > ef:
                        result.pop()

        return result

    def _select_neighbors_heuristic(
        self, query: np.ndarray, candidates: List[Tuple[float, int]], M: int
    ) -> List[Tuple[float, int]]:
        candidates.sort()
        selected = []
        remaining = list(candidates)

        while remaining and len(selected) < M:
            best = remaining.pop(0)
            best_dist, best_id = best

            good = True
            for sel_dist, sel_id in selected:
                d = self.dist_func(self.vectors[best_id], self.vectors[sel_id])
                if d < best_dist:
                    good = False
                    break

            if good:
                selected.append(best)

        if len(selected) < M:
            for cand in candidates:
                if cand not in selected:
                    selected.append(cand)
                    if len(selected) >= M:
                        break

        return selected[:M]

    def insert(self, vector: np.ndarray) -> int:
        vector = np.asarray(vector, dtype=np.float32).ravel()
        if vector.shape[0] != self.dim:
            raise ValueError(f"Expected dimension {self.dim}, got {vector.shape[0]}")

        node_id = len(self.vectors)
        self.vectors.append(vector)

        node_level = self._random_level()
        self.node_layers.append(node_level)

        while len(self.graphs) <= node_level:
            self.graphs.append({})

        for layer in range(node_level + 1):
            self.graphs[layer][node_id] = set()

        if self.entry_point is None:
            self.entry_point = node_id
            self.max_layer = node_level
            return node_id

        ep = [self.entry_point]
        current_layer = self.max_layer

        while current_layer > node_level:
            result = self._search_layer(vector, ep, ef=1, layer=current_layer)
            ep = [result[0][1]]
            current_layer -= 1

        for layer in range(min(node_level, self.max_layer), -1, -1):
            result = self._search_layer(vector, ep, ef=self.ef_construction, layer=layer)

            M_layer = self.M_max0 if layer == 0 else self.M
            neighbors = self._select_neighbors_heuristic(vector, result, M_layer)

            for dist, neighbor_id in neighbors:
                self.graphs[layer][node_id].add(neighbor_id)
                self.graphs[layer].setdefault(neighbor_id, set()).add(node_id)

                neighbor_conns = self.graphs[layer][neighbor_id]
                if len(neighbor_conns) > M_layer:
                    cands = [
                        (self.dist_func(self.vectors[neighbor_id], self.vectors[n]), n)
                        for n in neighbor_conns
                    ]
                    new_neighbors = self._select_neighbors_heuristic(
                        self.vectors[neighbor_id], cands, M_layer
                    )
                    new_set = {n_id for _, n_id in new_neighbors}
                    for old_n in neighbor_conns - new_set:
                        self.graphs[layer][old_n].discard(neighbor_id)
                    self.graphs[layer][neighbor_id] = new_set

            ep = [r[1] for r in result]

        if node_level > self.max_layer:
            self.entry_point = node_id
            self.max_layer = node_level

        return node_id

    def batch_insert(self, vectors: np.ndarray) -> List[int]:
        vectors = np.asarray(vectors, dtype=np.float32)
        if vectors.ndim == 1:
            vectors = vectors.reshape(1, -1)
        if vectors.shape[1] != self.dim:
            raise ValueError(f"Expected dimension {self.dim}, got {vectors.shape[1]}")

        ids = []
        for vec in vectors:
            ids.append(self.insert(vec))
        return ids

    def search(
        self, query: np.ndarray, k: int = 10, ef_search: Optional[int] = None
    ) -> Tuple[np.ndarray, np.ndarray]:
        if self.entry_point is None:
            return np.array([]), np.array([], dtype=int)

        query = np.asarray(query, dtype=np.float32).ravel()
        if query.shape[0] != self.dim:
            raise ValueError(f"Expected dimension {self.dim}, got {query.shape[0]}")

        ef = ef_search if ef_search is not None else self.ef_search
        ef = max(ef, k)

        ep = [self.entry_point]
        for layer in range(self.max_layer, 0, -1):
            result = self._search_layer(query, ep, ef=1, layer=layer)
            ep = [result[0][1]]

        result = self._search_layer(query, ep, ef=ef, layer=0)

        result.sort()
        result = result[:k]

        if not result:
            return np.array([]), np.array([], dtype=int)

        distances = np.array([r[0] for r in result])
        ids = np.array([r[1] for r in result], dtype=int)
        return distances, ids

    def batch_search(
        self, queries: np.ndarray, k: int = 10, ef_search: Optional[int] = None
    ) -> Tuple[np.ndarray, np.ndarray]:
        queries = np.asarray(queries, dtype=np.float32)
        if queries.ndim == 1:
            queries = queries.reshape(1, -1)

        all_distances = []
        all_ids = []
        for q in queries:
            d, i = self.search(q, k=k, ef_search=ef_search)
            if len(d) < k:
                d = np.pad(d, (0, k - len(d)), constant_values=float("inf"))
                i = np.pad(i, (0, k - len(i)), constant_values=-1)
            all_distances.append(d)
            all_ids.append(i)

        return np.array(all_distances), np.array(all_ids, dtype=int)

Benchmark：纯 Python 能打吗？

跑了一组对比测试，HNSW vs 暴力搜索，128 维向量，每组 100 次查询取平均：

Dataset: 1000 vectors, 128 dimensions, k=10
  Brute Force batch query: 10.3ms (100 queries), avg 0.103ms/query
  HNSW (ef_search=50) batch query: 104.6ms (100 queries), avg 1.046ms/query, Recall@10: 0.9850
  HNSW (ef_search=200) batch query: 291.9ms (100 queries), Recall@10: 1.0000

Dataset: 5000 vectors, 128 dimensions, k=10
  Brute Force batch query: 72.8ms (100 queries), avg 0.728ms/query
  HNSW (ef_search=50) batch query: 157.0ms (100 queries), avg 1.570ms/query, Recall@10: 0.8730
  HNSW (ef_search=200) batch query: 589.1ms (100 queries), Recall@10: 0.9950

Dataset: 10000 vectors, 128 dimensions, k=10
  Brute Force batch query: 148.3ms (100 queries), avg 1.483ms/query
  HNSW (ef_search=50) batch query: 173.2ms (100 queries), avg 1.732ms/query, Recall@10: 0.7420
  HNSW (ef_search=200) batch query: 727.1ms (100 queries), Recall@10: 0.9790

数字很诚实——纯 Python 实现的 HNSW 在这个量级上比暴力搜索还慢。

这完全在意料之中。暴力搜索那边，numpy 底层是 C 和 BLAS 在干活，矩阵乘法一步到位。而我的 HNSW 是纯 Python 循环，每次距离计算都有 Python 解释器的开销，图的遍历也全是字典查询和集合操作。

但看趋势：暴力搜索的耗时跟 N 成线性关系（1000→5000→10000，耗时大致是 10→73→148ms，接近线性）。HNSW 的增长明显缓慢得多（ef_search=50 时：105→157→173ms）。如果把数据量拉到 100K 甚至 1M，HNSW 的 O(log n) 复杂度优势就会碾压暴力搜索。

生产级别的库——hnswlib（C++）、FAISS（C++/CUDA）——在 100 万向量的规模上，单次查询延迟通常在亚毫秒级别，比暴力搜索快几个数量级。我这个纯 Python 版本的意义不在于性能，在于把算法拆开看清楚。

另一个有意思的观察是 ef_search 参数对 recall 和速度的影响。ef_search=50 时速度快但 recall 掉得厉害（10000 条向量时只有 0.742）；ef_search=200 时 recall 接近完美（0.979）但慢了 4 倍多。这就是 ANN 算法的核心 trade-off：你愿意用多少延迟换多少精度。实际部署的时候，这个参数要根据业务需求仔细调。
