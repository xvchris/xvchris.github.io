用 Rust 实现 BPE Tokenizer：从字节编码到 GPT-2 分词原理

⏱ 2026年3月15日
· 38 分钟阅读
#rust#ai#nlp#tokenizer#bpe

最近在用 LLM API 做一些文本处理，每次调用都会收到一个 token 数。费用按 token 算，但我对”token 到底是什么”一直停留在模糊的认知——大概是某种子词切分。

直到有一天我想在本地做 cost estimation，提前算出一段文本会被切成多少个 token，才发现自己搞不清楚 tokenizer 的具体逻辑。tiktoken 的代码翻了一遍，核心就是 BPE（Byte Pair Encoding）。那干脆自己用 Rust 写一个吧，跑通了就真懂了。

BPE 算法的本质

BPE 最初是一种数据压缩算法，Sennrich 等人在 2016 年把它引入了 NLP 做子词分割。GPT-2 在此基础上做了一个关键改动：字节级 BPE。

思路很直观。初始词表只有 256 个 token，对应 0x00 到 0xFF 每一个字节值。然后在训练语料上反复执行一个操作：

  1. 统计所有相邻 token 对的出现频率
  2. 找到频率最高的那对，合并成一个新 token
  3. 把这条合并规则记录下来
  4. 重复，直到词表大小达到目标值

比如语料里 t 和 h 经常相邻出现，第一轮合并可能就产生 th 这个新 token（id = 256）。下一轮 th 和 e 频率最高，合并出 the（id = 257）。以此类推。

字节级的好处在于：任何文本都能编码。不管是 ASCII、中文 UTF-8、emoji 还是二进制垃圾，都可以退化为单字节 token 的序列。不存在 OOV（out of vocabulary）问题。

实现里的几个设计决策

数据结构

BpeTokenizer 核心就三样东西：

  • vocab: HashMap<Vec<u8>, u32> — token 字节序列到 id 的映射
  • vocab_inverse: HashMap<u32, Vec<u8>> — 反向映射，decode 用
  • merges: Vec<(Vec<u8>, Vec<u8>)> — 有序的合并规则列表

合并规则的顺序很重要。encode 的时候要按训练时的顺序依次回放每条 merge，这样才能重现训练过程的分词结果。

Pre-tokenization

GPT-2 的做法是先按空格做一次粗切分，空格跟在后一个词的前面。"hello world" 会变成 ["hello", " world"]，注意 world 前面带了个空格。这个设计让模型能区分词首和词中的相同字符序列。

fn pre_tokenize(text: &str) -> Vec<String> {
    if text.is_empty() {
        return Vec::new();
    }

    let mut words = Vec::new();
    let mut current = String::new();

    for ch in text.chars() {
        if ch == ' ' && !current.is_empty() {
            words.push(current);
            current = String::new();
            current.push(ch); // 空格归到下一个词
        } else {
            current.push(ch);
        }
    }

    if !current.is_empty() {
        words.push(current);
    }

    words
}

我这里做了简化，GPT-2 实际用的是一个更复杂的正则表达式来处理标点、数字等边界。但核心思路一样。

训练：pair counting 循环

训练阶段的结构是个大循环。每一轮遍历所有词的 token 序列，统计相邻 pair 的频率，选出最高频的那对做合并。

一个容易忽略的优化点：语料里同一个词可能出现很多次，所以我先做了词频统计，pair count 的时候乘以词频，而不是把每次出现都展开。

// 词频预统计
let mut word_freqs: Vec<(Vec<Vec<u8>>, u64)> = Vec::new();
let mut word_index: HashMap<String, usize> = HashMap::new();

for word in &words {
    if let Some(&idx) = word_index.get(word) {
        word_freqs[idx].1 += 1;
    } else {
        let tokens: Vec<Vec<u8>> = word.bytes().map(|b| vec![b]).collect();
        let idx = word_freqs.len();
        word_index.insert(word.clone(), idx);
        word_freqs.push((tokens, 1));
    }
}

每次 merge 之后，要更新所有词的 token 序列。apply_merge 函数扫一遍 token 数组，把匹配的相邻 pair 替换成合并后的新 token：

fn apply_merge(tokens: &[Vec<u8>], left: &[u8], right: &[u8], merged: &[u8]) -> Vec<Vec<u8>> {
    if tokens.len() < 2 {
        return tokens.to_vec();
    }

    let mut result = Vec::with_capacity(tokens.len());
    let mut i = 0;

    while i < tokens.len() {
        if i + 1 < tokens.len() && tokens[i] == left && tokens[i + 1] == right {
            result.push(merged.to_vec());
            i += 2;
        } else {
            result.push(tokens[i].clone());
            i += 1;
        }
    }

    result
}

Encode 和 Decode

encode 就是把 pre-tokenize 的每个词拆成单字节 token，然后按 merges 的顺序依次回放合并。decode 更简单，把每个 id 查表拿到字节序列，拼起来转 UTF-8。

这种”按顺序回放 merge”的编码方式效率不是最优的——每条 merge 规则都要扫一遍当前 token 序列。但它保证了和训练结果一致，实现也简单。生产级的 tiktoken 用了更聪明的数据结构来加速这个过程。

序列化踩了个坑

写完核心逻辑想做持久化的时候，我顺手用 serde_json 去序列化整个 BpeTokenizer，直接报错了。

问题在 HashMap<Vec<u8>, u32>。JSON 的 key 必须是字符串，而 Vec<u8> 不是字符串。serde_json 在序列化 map 的时候会要求 key 实现某种到 string 的转换，Vec<u8> 不行。

想了想，其实没必要序列化整个 vocab。vocab 可以从 merge rules 完整重建——先初始化 256 个 base token，然后按顺序回放每条 merge 就能恢复全部 vocab。

所以最终只序列化 merge 规则：

#[derive(Serialize, Deserialize)]
struct BpeTokenizerData {
    merges: Vec<(Vec<u8>, Vec<u8>)>,
}

pub fn to_json(&self) -> Result<String, serde_json::Error> {
    let data = BpeTokenizerData {
        merges: self.merges.clone(),
    };
    serde_json::to_string_pretty(&data)
}

pub fn from_json(json: &str) -> Result<Self, serde_json::Error> {
    let data: BpeTokenizerData = serde_json::from_str(json)?;
    let mut tok = BpeTokenizer::new();
    for (left, right) in &data.merges {
        let mut merged = left.clone();
        merged.extend_from_slice(right);
        let new_id = tok.next_id;
        tok.vocab.insert(merged.clone(), new_id);
        tok.vocab_inverse.insert(new_id, merged);
        tok.merges.push((left.clone(), right.clone()));
        tok.next_id += 1;
    }
    Ok(tok)
}

这个方案还有个好处：序列化产物更小，只存了 delta 信息。

src/lib.rs

use serde::{Deserialize, Serialize};
use std::collections::HashMap;

#[derive(Debug, Clone)]
pub struct BpeTokenizer {
    vocab: HashMap<Vec<u8>, u32>,
    vocab_inverse: HashMap<u32, Vec<u8>>,
    merges: Vec<(Vec<u8>, Vec<u8>)>,
    next_id: u32,
}

#[derive(Serialize, Deserialize)]
struct BpeTokenizerData {
    merges: Vec<(Vec<u8>, Vec<u8>)>,
}

impl Default for BpeTokenizer {
    fn default() -> Self {
        Self::new()
    }
}

impl BpeTokenizer {
    pub fn new() -> Self {
        let mut vocab = HashMap::new();
        let mut vocab_inverse = HashMap::new();

        for byte_val in 0u16..=255 {
            let token = vec![byte_val as u8];
            let id = byte_val as u32;
            vocab.insert(token.clone(), id);
            vocab_inverse.insert(id, token);
        }

        BpeTokenizer {
            vocab,
            vocab_inverse,
            merges: Vec::new(),
            next_id: 256,
        }
    }

    pub fn train(&mut self, corpus: &str, vocab_size: u32) {
        assert!(
            vocab_size >= 256,
            "vocab_size must be at least 256 (base byte tokens)"
        );

        let words = pre_tokenize(corpus);

        let mut word_freqs: Vec<(Vec<Vec<u8>>, u64)> = Vec::new();
        let mut word_index: HashMap<String, usize> = HashMap::new();

        for word in &words {
            let key = word.clone();
            if let Some(&idx) = word_index.get(&key) {
                word_freqs[idx].1 += 1;
            } else {
                let tokens: Vec<Vec<u8>> = word.bytes().map(|b| vec![b]).collect();
                let idx = word_freqs.len();
                word_index.insert(key, idx);
                word_freqs.push((tokens, 1));
            }
        }

        let num_merges = vocab_size.saturating_sub(256);

        for _ in 0..num_merges {
            let mut pair_counts: HashMap<(Vec<u8>, Vec<u8>), u64> = HashMap::new();

            for (tokens, freq) in &word_freqs {
                if tokens.len() < 2 {
                    continue;
                }
                for pair in tokens.windows(2) {
                    let key = (pair[0].clone(), pair[1].clone());
                    *pair_counts.entry(key).or_insert(0) += freq;
                }
            }

            if pair_counts.is_empty() {
                break;
            }

            let best_pair = pair_counts
                .into_iter()
                .max_by_key(|&(_, count)| count)
                .unwrap()
                .0;

            let mut merged = best_pair.0.clone();
            merged.extend_from_slice(&best_pair.1);

            let new_id = self.next_id;
            self.vocab.insert(merged.clone(), new_id);
            self.vocab_inverse.insert(new_id, merged.clone());
            self.merges.push(best_pair.clone());
            self.next_id += 1;

            for (tokens, _) in word_freqs.iter_mut() {
                *tokens = apply_merge(tokens, &best_pair.0, &best_pair.1, &merged);
            }
        }
    }

    pub fn encode(&self, text: &str) -> Vec<u32> {
        let words = pre_tokenize(text);
        let mut all_ids = Vec::new();

        for word in &words {
            let mut tokens: Vec<Vec<u8>> = word.bytes().map(|b| vec![b]).collect();

            for (left, right) in &self.merges {
                let mut merged = left.clone();
                merged.extend_from_slice(right);
                tokens = apply_merge(&tokens, left, right, &merged);
            }

            for token in &tokens {
                if let Some(&id) = self.vocab.get(token) {
                    all_ids.push(id);
                } else {
                    for &b in token {
                        all_ids.push(b as u32);
                    }
                }
            }
        }

        all_ids
    }

    pub fn decode(&self, ids: &[u32]) -> Option<String> {
        let mut bytes = Vec::new();
        for &id in ids {
            let token_bytes = self.vocab_inverse.get(&id)?;
            bytes.extend_from_slice(token_bytes);
        }
        String::from_utf8(bytes).ok()
    }

    pub fn decode_bytes(&self, ids: &[u32]) -> Option<Vec<u8>> {
        let mut bytes = Vec::new();
        for &id in ids {
            let token_bytes = self.vocab_inverse.get(&id)?;
            bytes.extend_from_slice(token_bytes);
        }
        Some(bytes)
    }

    pub fn vocab_size(&self) -> u32 {
        self.next_id
    }

    pub fn num_merges(&self) -> usize {
        self.merges.len()
    }

    pub fn to_json(&self) -> Result<String, serde_json::Error> {
        let data = BpeTokenizerData {
            merges: self.merges.clone(),
        };
        serde_json::to_string_pretty(&data)
    }

    pub fn from_json(json: &str) -> Result<Self, serde_json::Error> {
        let data: BpeTokenizerData = serde_json::from_str(json)?;
        let mut tok = BpeTokenizer::new();
        for (left, right) in &data.merges {
            let mut merged = left.clone();
            merged.extend_from_slice(right);
            let new_id = tok.next_id;
            tok.vocab.insert(merged.clone(), new_id);
            tok.vocab_inverse.insert(new_id, merged);
            tok.merges.push((left.clone(), right.clone()));
            tok.next_id += 1;
        }
        Ok(tok)
    }

    pub fn id_to_token(&self, id: u32) -> Option<&[u8]> {
        self.vocab_inverse.get(&id).map(|v| v.as_slice())
    }

    pub fn token_to_id(&self, token: &[u8]) -> Option<u32> {
        self.vocab.get(token).copied()
    }
}

fn pre_tokenize(text: &str) -> Vec<String> {
    if text.is_empty() {
        return Vec::new();
    }

    let mut words = Vec::new();
    let mut current = String::new();

    for ch in text.chars() {
        if ch == ' ' && !current.is_empty() {
            words.push(current);
            current = String::new();
            current.push(ch);
        } else {
            current.push(ch);
        }
    }

    if !current.is_empty() {
        words.push(current);
    }

    words
}

fn apply_merge(tokens: &[Vec<u8>], left: &[u8], right: &[u8], merged: &[u8]) -> Vec<Vec<u8>> {
    if tokens.len() < 2 {
        return tokens.to_vec();
    }

    let mut result = Vec::with_capacity(tokens.len());
    let mut i = 0;

    while i < tokens.len() {
        if i + 1 < tokens.len() && tokens[i] == left && tokens[i + 1] == right {
            result.push(merged.to_vec());
            i += 2;
        } else {
            result.push(tokens[i].clone());
            i += 1;
        }
    }

    result
}

几个数字比较有意思。

decode 非常快，短文本 364 ns，因为它就是个查表拼接操作，没有任何计算量。encode 则慢得多，短文本也要 104 µs——因为每条 merge 规则都要扫一遍 token 序列，merge 规则越多越慢。这也是为什么 tiktoken 要用更高效的数据结构，真正的 GPT-2 有 50257 个 token，按我这个实现的方式线性回放 5 万条 merge 规则，性能会很糟糕。

训练速度方面，300 词表 1.3 ms、400 词表 2.8 ms，复杂度大致和 merge 次数成正比。每轮都要遍历全部词的 token 序列统计 pair 频率，数据量大的时候这会是瓶颈。

序列化和反序列化都在微秒级，这个不意外，merge 规则本身数据量很小。

和生产实现的差距

这个实现能帮助理解 BPE 的原理，但离生产使用还差不少。主要差在：

encode 性能。tiktoken 用了 trie 或 hash 做高效的 token 匹配，不需要逐条回放 merge。我这里是 O(merges × sequence_length)，tiktoken 可以做到接近 O(sequence_length)。

pre-tokenization。GPT-2 用的正则 's|'t|'re|'ve|'m|'ll|'d| ?\p{L}+| ?\p{N}+| ?[^\s\p{L}\p{N}]+|\s+ 比我的按空格分割精细得多，能更好地处理英语缩写和标点。

并行训练。pair counting 这步天然可以并行，用 rayon 拆分 word_freqs 再归并就行。我没加，因为这个实现重点是清晰。

不过作为一个”把原理跑通”的项目，这些都不是问题。搞清楚 token 是怎么回事比什么都重要。调 LLM API 的时候，至少我现在知道那个 token 数是怎么算出来的了。
