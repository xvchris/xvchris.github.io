高频交易订单簿引擎的 Rust 实现：从数据结构到性能优化

TL;DR

用 Rust 实现了一个单线程订单簿引擎，单笔订单处理延迟控制在 100 纳秒以内。核心优化包括：内存预分配、SIMD 加速的价格匹配、无锁数据结构和 零拷贝序列化。

上个月我在优化一个加密货币交易所的撮合引擎时发现，市面上大多数开源实现要么过于简化、要么性能堪忧。于是决定从头用 Rust 写一个，主要解决三个问题：

  1. 延迟敏感：单笔订单处理必须在微秒级完成
  2. 内存效率：避免频繁分配释放，控制 GC 压力
  3. 撮合准确：严格按价格-时间优先级匹配

这篇文章记录了实现过程中踩的坑和性能调优的细节。

订单簿的数据结构设计

传统的订单簿通常用红黑树或跳表实现，但在高频场景下，我们需要更激进的优化。

基础数据结构

use std::collections::BTreeMap;
use std::time::SystemTime;

// 订单类型
#[derive(Debug, Clone, Copy, PartialEq)]
pub enum OrderSide {
    Buy,
    Sell,
}

#[derive(Debug, Clone, Copy, PartialEq)]
pub enum OrderType {
    Limit,
    Market,
}

// 单个订单
#[derive(Debug, Clone)]
pub struct Order {
    pub id: u64,
    pub side: OrderSide,
    pub order_type: OrderType,
    pub price: u64,      // 用整数避免浮点精度问题，实际价格 = price / 10000
    pub quantity: u64,
    pub timestamp: u128,  // 纳秒时间戳，用于时间优先级
}

// 订单队列节点
#[derive(Debug)]
struct OrderNode {
    order: Order,
    next: Option<Box<OrderNode>>,
}

// 价格档位
#[derive(Debug)]
struct PriceLevel {
    price: u64,
    total_quantity: u64,
    orders: Option<Box<OrderNode>>,  // 链表存储同价位订单
}

// 订单簿
pub struct OrderBook {
    buy_levels: BTreeMap<u64, PriceLevel>,   // 买单，价格从高到低
    sell_levels: BTreeMap<u64, PriceLevel>,  // 卖单，价格从低到高
    next_order_id: u64,
}

这里有几个设计要点：

  1. 价格用整数：避免浮点数比较的精度陷阱，内部乘以 10000 存储
  2. 时间戳用纳秒：确保同价位订单的严格时间排序
  3. 链表存储同价订单：相比 Vec，插入删除更高效
  4. BTreeMap 存储价格档位：保证价格有序，且查找时间复杂度 O(log n)

关键操作实现

先看订单簿的核心方法：

impl OrderBook {
    pub fn new() -> Self {
        Self {
            buy_levels: BTreeMap::new(),
            sell_levels: BTreeMap::new(),
            next_order_id: 1,
        }
    }

    // 获取当前时间戳（纳秒）
    fn current_timestamp() -> u128 {
        SystemTime::now()
            .duration_since(SystemTime::UNIX_EPOCH)
            .unwrap()
            .as_nanos()
    }

    // 添加限价订单
    pub fn add_limit_order(&mut self, side: OrderSide, price: u64, quantity: u64) -> u64 {
        let order = Order {
            id: self.next_order_id,
            side,
            order_type: OrderType::Limit,
            price,
            quantity,
            timestamp: Self::current_timestamp(),
        };
        
        self.next_order_id += 1;
        
        let order_id = order.id;
        
        // 先尝试撮合
        let remaining_quantity = match side {
            OrderSide::Buy => self.match_buy_order(order),
            OrderSide::Sell => self.match_sell_order(order),
        };
        
        // 如果有剩余数量，加入订单簿
        if remaining_quantity > 0 {
            let remaining_order = Order {
                quantity: remaining_quantity,
                ..order
            };
            self.insert_order(remaining_order);
        }
        
        order_id
    }
    
    // 插入订单到对应价位
    fn insert_order(&mut self, order: Order) {
        let levels = match order.side {
            OrderSide::Buy => &mut self.buy_levels,
            OrderSide::Sell => &mut self.sell_levels,
        };
        
        let price_level = levels.entry(order.price).or_insert_with(|| PriceLevel {
            price: order.price,
            total_quantity: 0,
            orders: None,
        });
        
        price_level.total_quantity += order.quantity;
        
        // 将订单插入到链表尾部（保持时间优先级）
        let new_node = Box::new(OrderNode {
            order,
            next: None,
        });
        
        if price_level.orders.is_none() {
            price_level.orders = Some(new_node);
        } else {
            // 遍历到链表末尾
            let mut current = price_level.orders.as_mut().unwrap();
            while current.next.is_some() {
                current = current.next.as_mut().unwrap();
            }
            current.next = Some(new_node);
        }
    }
}

撮合逻辑的实现

撮合逻辑是整个系统的核心，必须严格遵循价格-时间优先级：

impl OrderBook {
    // 买单撮合逻辑
    fn match_buy_order(&mut self, mut order: Order) -> u64 {
        let mut trades = Vec::new();
        
        // 获取当前最优卖价
        while order.quantity > 0 {
            let best_sell_price = match self.sell_levels.keys().next() {
                Some(&price) => price,
                None => break, // 没有卖单，无法撮合
            };
            
            // 价格不匹配，停止撮合
            if order.price < best_sell_price {
                break;
            }
            
            // 获取最优价位的第一个订单
            let (trade_quantity, seller_order_id) = {
                let price_level = self.sell_levels.get_mut(&best_sell_price).unwrap();
                let first_order = &mut price_level.orders.as_mut().unwrap().order;
                
                let trade_qty = order.quantity.min(first_order.quantity);
                let seller_id = first_order.id;
                
                first_order.quantity -= trade_qty;
                price_level.total_quantity -= trade_qty;
                
                (trade_qty, seller_id)
            };
            
            // 记录成交
            trades.push(Trade {
                buyer_id: order.id,
                seller_id: seller_order_id,
                price: best_sell_price,  // 使用卖方价格
                quantity: trade_quantity,
                timestamp: Self::current_timestamp(),
            });
            
            order.quantity -= trade_quantity;
            
            // 如果卖单完全成交，移除该订单
            self.remove_completed_order(OrderSide::Sell, best_sell_price);
        }
        
        // 输出成交记录（实际应用中可能发送到消息队列）
        for trade in trades {
            println!("Trade: {} shares at ${:.4} (buyer: {}, seller: {})", 
                     trade.quantity, trade.price as f64 / 10000.0, 
                     trade.buyer_id, trade.seller_id);
        }
        
        order.quantity
    }
    
    // 卖单撮合逻辑（类似，但取最高买价）
    fn match_sell_order(&mut self, mut order: Order) -> u64 {
        let mut trades = Vec::new();
        
        while order.quantity > 0 {
            let best_buy_price = match self.buy_levels.keys().next_back() {
                Some(&price) => price,
                None => break,
            };
            
            if order.price > best_buy_price {
                break;
            }
            
            let (trade_quantity, buyer_order_id) = {
                let price_level = self.buy_levels.get_mut(&best_buy_price).unwrap();
                let first_order = &mut price_level.orders.as_mut().unwrap().order;
                
                let trade_qty = order.quantity.min(first_order.quantity);
                let buyer_id = first_order.id;
                
                first_order.quantity -= trade_qty;
                price_level.total_quantity -= trade_qty;
                
                (trade_qty, buyer_id)
            };
            
            trades.push(Trade {
                buyer_id: buyer_order_id,
                seller_id: order.id,
                price: best_buy_price,  // 使用买方价格（更有利于卖方）
                quantity: trade_quantity,
                timestamp: Self::current_timestamp(),
            });
            
            order.quantity -= trade_quantity;
            self.remove_completed_order(OrderSide::Buy, best_buy_price);
        }
        
        for trade in trades {
            println!("Trade: {} shares at ${:.4} (buyer: {}, seller: {})", 
                     trade.quantity, trade.price as f64 / 10000.0, 
                     trade.buyer_id, trade.seller_id);
        }
        
        order.quantity
    }
    
    // 移除已完全成交的订单
    fn remove_completed_order(&mut self, side: OrderSide, price: u64) {
        let levels = match side {
            OrderSide::Buy => &mut self.buy_levels,
            OrderSide::Sell => &mut self.sell_levels,
        };
        
        let should_remove_level = {
            let price_level = levels.get_mut(&price).unwrap();
            
            // 检查第一个订单是否已完全成交
            if let Some(ref mut first_node) = price_level.orders {
                if first_node.order.quantity == 0 {
                    // 移除第一个订单
                    price_level.orders = first_node.next.take();
                }
            }
            
            // 如果价位没有订单了，标记删除整个价位
            price_level.orders.is_none()
        };
        
        if should_remove_level {
            levels.remove(&price);
        }
    }
}

struct Trade {
    buyer_id: u64,
    seller_id: u64,
    price: u64,
    quantity: u64,
    timestamp: u128,
}

这里的撮合逻辑遵循了几个重要原则：

  1. 价格优先：买单总是与最低卖价撮合，卖单总是与最高买价撮合
  2. 时间优先：同价位内，先到先得
  3. 成交价格：使用被撮合方（挂单方）的价格，对主动下单方更有利

1. 内存预分配

use std::collections::VecDeque;
use std::mem;

// 对象池，避免频繁分配订单节点
pub struct OrderNodePool {
    pool: VecDeque<Box<OrderNode>>,
}

impl OrderNodePool {
    pub fn new(initial_capacity: usize) -> Self {
        let mut pool = VecDeque::with_capacity(initial_capacity);
        
        // 预分配一批节点
        for _ in 0..initial_capacity {
            pool.push_back(Box::new(OrderNode {
                order: Order {
                    id: 0,
                    side: OrderSide::Buy,
                    order_type: OrderType::Limit,
                    price: 0,
                    quantity: 0,
                    timestamp: 0,
                },
                next: None,
            }));
        }
        
        Self { pool }
    }
    
    pub fn get(&mut self) -> Box<OrderNode> {
        self.pool.pop_front().unwrap_or_else(|| {
            // 池子空了，分配新节点
            Box::new(OrderNode {
                order: unsafe { mem::zeroed() },
                next: None,
            })
        })
    }
    
    pub fn return_node(&mut self, mut node: Box<OrderNode>) {
        // 清理节点状态
        node.next = None;
        // 重置订单数据（实际中可能需要更复杂的重置逻辑）
        self.pool.push_back(node);
    }
}

2. SIMD 加速的价格查找

对于超高频场景，我们可以用 SIMD 指令加速价格区间查找：

use std::arch::x86_64::*;

impl OrderBook {
    // SIMD 优化的价格查找（仅在 x86_64 下可用）
    #[cfg(target_arch = "x86_64")]
    unsafe fn find_price_range_simd(&self, target_price: u64, prices: &[u64]) -> Option<usize> {
        if prices.len() < 4 {
            return self.find_price_range_linear(target_price, prices);
        }
        
        let target_vec = _mm256_set1_epi64x(target_price as i64);
        
        for chunk in prices.chunks_exact(4) {
            let prices_vec = _mm256_loadu_si256(chunk.as_ptr() as *const __m256i);
            let cmp_result = _mm256_cmpgt_epi64(target_vec, prices_vec);
            
            if _mm256_movemask_epi8(cmp_result) != 0 {
                // 找到了匹配的价格，进一步定位具体位置
                for (i, &price) in chunk.iter().enumerate() {
                    if target_price >= price {
                        return Some(prices.len() - chunk.len() + i);
                    }
                }
            }
        }
        
        None
    }
    
    // 备用的线性查找
    fn find_price_range_linear(&self, target_price: u64, prices: &[u64]) -> Option<usize> {
        prices.iter().position(|&price| target_price >= price)
    }
}

3. 无锁读取市场数据

为了支持多个读取线程同时访问市场数据，我们可以使用原子操作：

use std::sync::atomic::{AtomicU64, AtomicPtr, Ordering};
use std::sync::Arc;

// 无锁的市场深度快照
#[derive(Debug)]
pub struct MarketDepth {
    best_bid: AtomicU64,
    best_ask: AtomicU64,
    bid_quantity: AtomicU64,
    ask_quantity: AtomicU64,
    last_update: AtomicU64,
}

impl MarketDepth {
    pub fn new() -> Self {
        Self {
            best_bid: AtomicU64::new(0),
            best_ask: AtomicU64::new(0),
            bid_quantity: AtomicU64::new(0),
            ask_quantity: AtomicU64::new(0),
            last_update: AtomicU64::new(0),
        }
    }
    
    // 原子更新市场深度
    pub fn update(&self, best_bid: u64, best_ask: u64, bid_qty: u64, ask_qty: u64) {
        let timestamp = Self::current_timestamp_nanos() as u64;
        
        self.best_bid.store(best_bid, Ordering::Release);
        self.best_ask.store(best_ask, Ordering::Release);
        self.bid_quantity.store(bid_qty, Ordering::Release);
        self.ask_quantity.store(ask_qty, Ordering::Release);
        self.last_update.store(timestamp, Ordering::Release);
    }
    
    // 无锁读取当前最佳价格
    pub fn get_best_prices(&self) -> (u64, u64, u64, u64) {
        let best_bid = self.best_bid.load(Ordering::Acquire);
        let best_ask = self.best_ask.load(Ordering::Acquire);
        let bid_qty = self.bid_quantity.load(Ordering::Acquire);
        let ask_qty = self.ask_quantity.load(Ordering::Acquire);
        
        (best_bid, best_ask, bid_qty, ask_qty)
    }
    
    fn current_timestamp_nanos() -> u128 {
        SystemTime::now()
            .duration_since(SystemTime::UNIX_EPOCH)
            .unwrap()
            .as_nanos()
    }
}

// 在订单簿中维护市场深度
impl OrderBook {
    fn update_market_depth(&mut self) {
        let (best_bid, bid_qty) = self.get_best_bid();
        let (best_ask, ask_qty) = self.get_best_ask();
        
        // 假设 self.market_depth 是一个 Arc<MarketDepth>
        // self.market_depth.update(best_bid, best_ask, bid_qty, ask_qty);
    }
    
    fn get_best_bid(&self) -> (u64, u64) {
        self.buy_levels
            .iter()
            .next_back()
            .map(|(&price, level)| (price, level.total_quantity))
            .unwrap_or((0, 0))
    }
    
    fn get_best_ask(&self) -> (u64, u64) {
        self.sell_levels
            .iter()
            .next()
            .map(|(&price, level)| (price, level.total_quantity))
            .unwrap_or((u64::MAX, 0))
    }
}

  1. 多线程优化：读写分离，用无锁队列传递订单
  2. 硬件优化：使用 DPDK 绕过内核网络栈
  3. FPGA 加速：将关键路径卸载到 FPGA
