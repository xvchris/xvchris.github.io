用 Rust 实现 GARCH(1,1) 波动率模型：从公式推导到动态仓位管理

⏱ 2026年3月17日
· 20 分钟阅读
#rust#quantitative-finance#volatility#risk-management

上周写配对交易和执行算法时，几乎每篇都绕不开”当前波动率”这个输入。我一直在用一个简单的 20 日滚动标准差凑合，但波动率本身会聚集——大波动之后往往还是大波动，这个现象滚动窗口抓不住。

GARCH(1,1) 是处理这个问题最经典的工具。公式不复杂，实现也不依赖什么特殊 crate，正好拿 Rust 来写一遍，顺带接上动态仓位管理的逻辑。

测试环境：macOS 15.7.3，arm64，rustc 1.94.0 (4a4ef493e 2026-03-02)，cargo 1.94.0 (85eff7c80 2026-01-15)。

GARCH 是什么，为什么比滚动窗口好

GARCH(1,1) 的核心方程：

σ²_t = ω + α · r²_{t-1} + β · σ²_{t-1}

三个参数的直觉：

  • ω（omega）：长期基准方差，保证模型不会跌到零
  • α（alpha）：对上一期冲击（r²_{t-1}）的反应速度，越大说明模型”反应越快”
  • β（beta）：方差的持续性，越大说明高/低波动状态越难消退

α + β < 1 是弱平稳的必要条件，这时存在无条件（长期）方差：

σ²_∞ = ω / (1 - α - β)

对比 20 日滚动标准差：滚动窗口对每天赋予相等权重，而 GARCH 对近期冲击赋予更高权重。更重要的是，GARCH 可以做前向预测，滚动标准差只是对历史的平均，预测未来 k 天波动没有理论依据。

参数结构和验证

先把约束条件写进构造函数，让非法状态无法表达：

#[derive(Debug, Clone)]
pub struct GarchParams {
    pub omega: f64,
    pub alpha: f64,
    pub beta: f64,
}

impl GarchParams {
    pub fn new(omega: f64, alpha: f64, beta: f64) -> Result<Self, &'static str> {
        if omega <= 0.0 {
            return Err("omega must be positive");
        }
        if alpha < 0.0 || beta < 0.0 {
            return Err("alpha and beta must be non-negative");
        }
        if alpha + beta >= 1.0 {
            return Err("alpha + beta must be < 1.0 (covariance stationarity)");
        }
        Ok(GarchParams { omega, alpha, beta })
    }

    pub fn unconditional_variance(&self) -> f64 {
        self.omega / (1.0 - self.alpha - self.beta)
    }

    pub fn unconditional_vol_annualized(&self) -> f64 {
        self.unconditional_variance().sqrt() * 252.0_f64.sqrt()
    }
}

α + β ≥ 1 的情况叫做 IGARCH（积分 GARCH），方差有单位根，波动率冲击永不消散。这在某些高频金融数据上确实出现，但本文不处理这种边缘情况。

递推滤波

给定参数和一段历史收益率，逐日计算条件方差：

pub fn garch_filter(params: &GarchParams, returns: &[f64]) -> Vec<f64> {
    if returns.is_empty() {
        return vec![];
    }

    let mut variances = Vec::with_capacity(returns.len());
    // 用无条件方差初始化，避免 burn-in 时序偏差
    let mut h = params.unconditional_variance();

    for &r in returns {
        h = params.omega + params.alpha * r * r + params.beta * h;
        variances.push(h);
    }
    variances
}

初始化方式有几种选择：可以用样本方差，可以用参数隐含的无条件方差，也可以固定一个小值然后扔掉前几十天的 burn-in 期。我这里用无条件方差，省掉了 burn-in 的麻烦。

多步前向预测

GARCH(1,1) 的 k 步预测有解析解：

E[σ²_{t+k}] = σ²_∞ + (α+β)^k · (σ²_t - σ²_∞)

这个公式很直观：当前偏离长期均值的部分，以 (α+β) 的速率指数衰减。α+β 越接近 1，衰减越慢，波动状态越”粘”。

pub fn forecast_variance(params: &GarchParams, current_variance: f64, horizon: usize) -> Vec<f64> {
    let persistence = params.alpha + params.beta;
    let long_run = params.unconditional_variance();
    let mut forecasts = Vec::with_capacity(horizon);

    let mut prev = current_variance;
    for _ in 1..=horizon {
        let fcast = long_run + persistence * (prev - long_run);
        forecasts.push(fcast);
        prev = fcast;
    }
    forecasts
}

参数估计：极大似然

实际使用中参数需要从数据里估出来。假设新息服从条件正态分布，负对数似然是：

L = 0.5 · Σ [ln(σ²_t) + r²_t / σ²_t]

生产中应该用 L-BFGS-B 之类的梯度方法，但写这篇文章我不想引入 argmin 或 nlopt，就用网格搜索凑合一下：

fn neg_log_likelihood(omega: f64, alpha: f64, beta: f64, returns: &[f64]) -> f64 {
    if omega <= 0.0 || alpha < 0.0 || beta < 0.0 || alpha + beta >= 1.0 {
        return f64::INFINITY;
    }
    let params = match GarchParams::new(omega, alpha, beta) {
        Ok(p) => p,
        Err(_) => return f64::INFINITY,
    };

    let h0 = params.unconditional_variance();
    let mut h = h0;
    let mut nll = 0.0;

    for &r in returns {
        h = omega + alpha * r * r + beta * h;
        if h <= 0.0 {
            return f64::INFINITY;
        }
        nll += 0.5 * (h.ln() + r * r / h);
    }
    nll
}

pub fn fit_garch_grid(returns: &[f64]) -> GarchParams {
    let mut best_nll = f64::INFINITY;
    let mut best_params = GarchParams { omega: 1e-6, alpha: 0.1, beta: 0.8 };

    let mean_sq_r = returns.iter().map(|r| r * r).sum::<f64>() / returns.len() as f64;

    let alpha_grid: Vec<f64> = (1..=15).map(|i| i as f64 * 0.02).collect();
    let beta_grid: Vec<f64> = (50..=92).map(|i| i as f64 * 0.01).collect();

    for &alpha in &alpha_grid {
        for &beta in &beta_grid {
            if alpha + beta >= 0.999 {
                continue;
            }
            // 给定 α、β，最优 ω 可以约束为样本方差乘以 (1-α-β)
            let omega = (1.0 - alpha - beta) * mean_sq_r;
            if omega <= 0.0 {
                continue;
            }
            let nll = neg_log_likelihood(omega, alpha, beta, returns);
            if nll < best_nll {
                best_nll = nll;
                best_params = GarchParams { omega, alpha, beta };
            }
        }
    }

    best_params
}

关于 ω 的这个推导：如果要求模型的无条件方差等于样本方差，那 ω = (1-α-β) · 样本方差，这样三个参数就剩两个需要搜索，大幅缩小了空间。

用 756 天模拟数据（真实参数 α=0.12, β=0.85）测试：

真实参数: GARCH(1,1)[ω=0.000002, α=0.1200, β=0.8500, α+β=0.9700]
拟合参数: GARCH(1,1)[ω=0.000001, α=0.3000, β=0.6900, α+β=0.9900]

α 和 β 的具体值偏差不小，但 α+β（持续性）和无条件方差都拟合得较准。这个现象在 GARCH 估计里很常见：似然函数在 α+β 接近 1 的区域非常平坦，导致 α 和 β 很难分辨，但它们的和比较稳定。

动态仓位管理

有了逐日条件波动率，最直接的应用是目标波动率仓位缩放：

仓位大小 = 目标波动率 / 当前波动率

直觉是高波动期降杠杆，低波动期加杠杆，维持组合波动率大致稳定。

pub struct VolTargetSizer {
    pub target_vol: f64,
    pub max_leverage: f64,
    pub min_leverage: f64,
}

impl VolTargetSizer {
    pub fn size(&self, current_daily_var: f64) -> f64 {
        let current_daily_vol = current_daily_var.sqrt();
        let current_annual_vol = current_daily_vol * 252.0_f64.sqrt();

        if current_annual_vol <= 0.0 {
            return self.max_leverage;
        }

        let raw = self.target_vol / current_annual_vol;
        raw.clamp(self.min_leverage, self.max_leverage)
    }
}

用前一天的方差决定今天的仓位，避免用当天信息（前视偏差）：

pub fn backtest(
    returns: &[f64],

模拟 756 天的结果：

策略收益: 26.6%  波动率: 16.8%  Sharpe: 0.55  最大回撤: 15.9%
基准收益: 14.2%  波动率: 13.2%  Sharpe: 0.10

这组模拟数据里策略效果明显，但要注意两点：第一，模拟数据的 seed 固定了，换 seed 结果差异可能很大；第二，交易成本完全没有计进去，实盘中每天调整仓位的磨损可能把 Sharpe 吃掉一大半。

几个实现细节

不依赖 rand crate：为了零依赖，自己写了一个线性同余生成器（LCG）加 Box-Muller 变换来生成正态分布。LCG 的质量在统计检验上不过关，但对演示够用了。真正用时换成 rand + rand_distr crate（需要在 Cargo.toml 加 rand = "0.8" 和 rand_distr = "0.4"）：

use rand_distr::{Distribution, StandardNormal};
let z: f64 = StandardNormal.sample(&mut rng);

精度问题：方差在日度数据上通常是 1e-4 到 1e-6 量级，f64 完全足够。但如果做微秒频率的高频数据，乘以 252 或开根号的精度损失要单独评估。

初始值敏感性：滤波器的第一步用 unconditional_variance() 初始化，这在数据量够多时没问题。如果数据很短（< 50 天），建议前几十天只做 burn-in 不进 likelihood 计算。

接下来

这篇只覆盖了最基础的 GARCH(1,1) 和最简单的动态仓位逻辑。实际上还有几个方向值得深入：

  • GJR-GARCH（又叫 TGARCH）：下跌冲击对波动率的影响比上涨更大（杠杆效应），GJR 用一个额外参数捕捉这个不对称性
  • 多变量 DCC-GARCH：组合里多个资产的条件相关矩阵随时间变化
  • VaR 和 CVaR：有了条件分布就可以做更准确的风险度量，不再依赖历史分位数
