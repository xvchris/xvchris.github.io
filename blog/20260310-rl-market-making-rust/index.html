强化学习做市策略的 Rust 实现：从 Q-Learning 到实盘部署

⏱ 2026年3月10日
· 55 分钟阅读
#rust#reinforcement-learning#market-making#trading

最近在研究如何用强化学习来优化做市策略，发现大部分开源实现都是 Python 写的，性能上很难满足实盘需求。做市需要在毫秒级别内做出报价决策，Python 的速度实在是个瓶颈。于是我用 Rust 重新实现了一套基于 Q-Learning 的做市智能体，性能提升了不少。

做市策略的核心问题

做市本质上是在买卖价差中赚钱，但面临几个核心挑战：

  1. 库存风险：持仓过多单边头寸会面临价格波动损失
  2. 选择时机：什么时候应该主动报价，什么时候应该撤单
  3. 价格设置：买单和卖单应该距离中间价多远
  4. 对手方识别：避免被知情交易者”吃掉”

传统做市策略通常用简单的规则（比如固定价差、库存管理阈值），但市场环境复杂多变，固定规则很难适应。强化学习可以让策略自动学习最优的报价行为。

强化学习框架设计

我选择了 Q-Learning 算法，主要考虑：

  • 离散动作空间：做市的动作相对简单（报买单/卖单/撤单）
  • 样本效率：相比 Policy Gradient，Q-Learning 对样本数量要求没那么高
  • 可解释性：可以直接查看 Q 表来理解策略偏好

状态空间定义

状态空间设计是关键。我定义了一个 MarketState 结构：

#[derive(Debug, Clone, Hash, Eq, PartialEq)]
pub struct MarketState {
    // 库存状态 (离散化为 -10 到 +10)
    pub inventory: i8,
    
    // 买卖价差 (bp 为单位，离散化)
    pub spread: u8,
    
    // 订单簿不平衡度 (-100 到 +100)
    pub imbalance: i8,
    
    // 短期价格动量 (-5 到 +5)
    pub momentum: i8,
    
    // 我们当前的报价状态
    pub quote_state: QuoteState,
}

#[derive(Debug, Clone, Hash, Eq, PartialEq)]
pub enum QuoteState {
    NoQuotes,           // 没有挂单
    BidOnly,            // 只有买单
    AskOnly,            // 只有卖单
    BothSides,          // 双边报价
}

状态空间维度控制在合理范围内（大约 10 * 20 * 200 * 10 * 4 = 1,600,000），既能捕获重要信息，又不至于过于稀疏。

动作空间设计

动作空间包含所有可能的报价决策：

#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub enum Action {
    // 不做任何操作
    DoNothing,
    
    // 报买单 (距离中间价的 tick 数)
    PlaceBid(u8),   // 1-5 ticks
    
    // 报卖单 (距离中间价的 tick 数)  
    PlaceAsk(u8),   // 1-5 ticks
    
    // 双边报价
    PlaceBoth { bid_dist: u8, ask_dist: u8 },
    
    // 撤单操作
    CancelBids,
    CancelAsks,
    CancelAll,
}

impl Action {
    pub fn all_actions() -> Vec<Action> {
        let mut actions = vec![Action::DoNothing, Action::CancelBids, 
                             Action::CancelAsks, Action::CancelAll];
        
        // 单边报价 (1-5 ticks)
        for dist in 1..=5 {
            actions.push(Action::PlaceBid(dist));
            actions.push(Action::PlaceAsk(dist));
        }
        
        // 双边报价 (1-3 ticks 的组合)
        for bid_dist in 1..=3 {
            for ask_dist in 1..=3 {
                actions.push(Action::PlaceBoth { bid_dist, ask_dist });
            }
        }
        
        actions
    }
}

总共 23 个动作（4 个基础操作 + 10 个单边报价 + 9 个双边报价组合），涵盖了做市的主要策略选择。

Q-Learning 算法实现

Q-Learning 的核心是 Q 表更新：Q(s,a) ← Q(s,a) + α[r + γ max Q(s’,a’) - Q(s,a)]

use std::collections::HashMap;
use rand::RngExt;

pub struct QLearningAgent {
    q_table: HashMap<(MarketState, Action), f64>,
    learning_rate: f64,
    discount_factor: f64,
    epsilon: f64,
    epsilon_decay: f64,
    min_epsilon: f64,
}

impl QLearningAgent {
    pub fn new(learning_rate: f64, discount_factor: f64, epsilon: f64) -> Self {
        Self {
            q_table: HashMap::new(),
            learning_rate,
            discount_factor,
            epsilon,
            epsilon_decay: 0.9995,
            min_epsilon: 0.01,
        }
    }
    
    pub fn choose_action(&mut self, state: &MarketState) -> Action {
        if rand::rng().random::<f64>() < self.epsilon {
            // ε-贪婪：随机探索
            self.random_action()
        } else {
            // 选择最优动作
            self.best_action(state)
        }
    }
    
    fn best_action(&self, state: &MarketState) -> Action {
        let actions = Action::all_actions();
        
        actions
            .into_iter()
            .max_by(|a, b| {
                let q_a = self.get_q_value(state, a);
                let q_b = self.get_q_value(state, b);
                q_a.partial_cmp(&q_b).unwrap()
            })
            .unwrap_or(Action::DoNothing)
    }
    
    fn random_action(&self) -> Action {
        let actions = Action::all_actions();
        let idx = rand::rng().random_range(0..actions.len());
        actions[idx]
    }
    
    fn get_q_value(&self, state: &MarketState, action: &Action) -> f64 {
        *self.q_table.get(&(state.clone(), *action)).unwrap_or(&0.0)
    }
    
    pub fn update_q_value(
        &mut self,
        state: &MarketState,
        action: &Action,
        reward: f64,
        next_state: &MarketState,
    ) {
        let current_q = self.get_q_value(state, action);
        let max_next_q = self.get_max_q_value(next_state);
        
        let new_q = current_q + self.learning_rate * 
            (reward + self.discount_factor * max_next_q - current_q);
        
        self.q_table.insert((state.clone(), *action), new_q);
        
        // ε 衰减
        self.epsilon = (self.epsilon * self.epsilon_decay).max(self.min_epsilon);
    }
    
    fn get_max_q_value(&self, state: &MarketState) -> f64 {
        Action::all_actions()
            .into_iter()
            .map(|a| self.get_q_value(state, &a))
            .fold(f64::NEG_INFINITY, f64::max)
    }
}

订单簿处理与环境模拟

做市策略需要实时处理订单簿数据，我实现了一个简化的订单簿：

use std::collections::BTreeMap;

#[derive(Debug, Clone)]
pub struct OrderBook {
    bids: BTreeMap<i64, f64>,  // price -> quantity
    asks: BTreeMap<i64, f64>,  // price -> quantity
}

impl OrderBook {
    pub fn new() -> Self {
        Self {
            bids: BTreeMap::new(),
            asks: BTreeMap::new(),
        }
    }
    
    pub fn best_bid(&self) -> Option<(i64, f64)> {
        self.bids.iter().next_back().map(|(&p, &q)| (p, q))
    }
    
    pub fn best_ask(&self) -> Option<(i64, f64)> {
        self.asks.iter().next().map(|(&p, &q)| (p, q))
    }
    
    pub fn mid_price(&self) -> Option<f64> {
        match (self.best_bid(), self.best_ask()) {
            (Some((bid, _)), Some((ask, _))) => Some((bid + ask) as f64 / 2.0),
            _ => None,
        }
    }
    
    pub fn spread(&self) -> Option<i64> {
        match (self.best_bid(), self.best_ask()) {
            (Some((bid, _)), Some((ask, _))) => Some(ask - bid),
            _ => None,
        }
    }
    
    pub fn imbalance(&self) -> f64 {
        let bid_volume: f64 = self.bids.values().take(5).sum();
        let ask_volume: f64 = self.asks.values().take(5).sum();
        
        if bid_volume + ask_volume == 0.0 {
            0.0
        } else {
            (bid_volume - ask_volume) / (bid_volume + ask_volume)
        }
    }
    
    pub fn add_order(&mut self, side: Side, price: i64, quantity: f64) {
        match side {
            Side::Buy => {
                *self.bids.entry(price).or_insert(0.0) += quantity;
            }
            Side::Sell => {
                *self.asks.entry(price).or_insert(0.0) += quantity;
            }
        }
    }
    
    pub fn remove_order(&mut self, side: Side, price: i64, quantity: f64) {
        match side {
            Side::Buy => {
                if let Some(q) = self.bids.get_mut(&price) {
                    *q -= quantity;
                    if *q <= 0.0 {
                        self.bids.remove(&price);
                    }
                }
            }
            Side::Sell => {
                if let Some(q) = self.asks.get_mut(&price) {
                    *q -= quantity;
                    if *q <= 0.0 {
                        self.asks.remove(&price);
                    }
                }
            }
        }
    }
}

#[derive(Debug, Clone, Copy, PartialEq)]
pub enum Side {
    Buy,
    Sell,
}

奖励函数设计

奖励函数是强化学习的核心，直接决定了策略的行为偏好：

pub struct RewardCalculator {
    inventory_penalty_factor: f64,
    spread_capture_bonus: f64,
    adverse_selection_penalty: f64,
}

impl RewardCalculator {
    pub fn new() -> Self {
        Self {
            inventory_penalty_factor: 0.01,
            spread_capture_bonus: 1.0,
            adverse_selection_penalty: -2.0,
        }
    }
    
    pub fn calculate_reward(
        &self,
        trade_pnl: f64,
        inventory_change: i32,
        current_inventory: i32,
        was_adverse_selection: bool,
    ) -> f64 {
        let mut reward = 0.0;
        
        // 1. 交易盈亏
        reward += trade_pnl;
        
        // 2. 价差捕获奖励
        if trade_pnl > 0.0 {
            reward += self.spread_capture_bonus;
        }
        
        // 3. 库存惩罚（二次函数，库存越极端惩罚越大）
        let inventory_penalty = self.inventory_penalty_factor * 
            (current_inventory as f64).powi(2);
        reward -= inventory_penalty;
        
        // 4. 逆向选择惩罚
        if was_adverse_selection {
            reward += self.adverse_selection_penalty;
        }
        
        // 5. 库存收敛奖励（库存朝零方向变化给予小奖励）
        if current_inventory.abs() > (current_inventory - inventory_change).abs() {
            reward += 0.1;
        }
        
        reward
    }
}

这个奖励函数鼓励：

  • 赚取价差（正向交易盈亏）
  • 保持库存平衡（避免极端头寸）
  • 避免被逆向选择（被知情交易者利用）

市场环境模拟器

为了训练智能体，我构建了一个简化的市场环境：

use rand_distr::{Distribution, Normal};

pub struct MarketEnvironment {
    orderbook: OrderBook,
    current_inventory: i32,
    last_mid_price: f64,
    price_volatility: f64,
    tick_size: f64,
    our_orders: Vec<Order>,
    reward_calculator: RewardCalculator,
}

#[derive(Debug, Clone)]
pub struct Order {
    pub id: u64,
    pub side: Side,
    pub price: i64,
    pub quantity: f64,
    pub timestamp: u64,
}

impl MarketEnvironment {
    pub fn new(initial_mid_price: f64, tick_size: f64, volatility: f64) -> Self {
        let mut env = Self {
            orderbook: OrderBook::new(),
            current_inventory: 0,
            last_mid_price: initial_mid_price,
            price_volatility: volatility,
            tick_size,
            our_orders: Vec::new(),
            reward_calculator: RewardCalculator::new(),
        };
        
        // 初始化订单簿
        env.initialize_orderbook();
        env
    }
    
    fn initialize_orderbook(&mut self) {
        let mid_ticks = (self.last_mid_price / self.tick_size) as i64;
        
        // 添加一些初始流动性
        for i in 1..=10 {
            let bid_price = mid_ticks - i;
            let ask_price = mid_ticks + i;
            let quantity = 100.0 / (i as f64);
            
            self.orderbook.add_order(Side::Buy, bid_price, quantity);
            self.orderbook.add_order(Side::Sell, ask_price, quantity);
        }
    }
    
    pub fn step(&mut self, action: Action) -> (MarketState, f64, bool) {
        // 1. 模拟市场价格变动
        self.simulate_market_move();
        
        // 2. 执行智能体动作
        let reward = self.execute_action(action);
        
        // 3. 检查我们的订单是否被成交
        self.check_order_fills();
        
        // 4. 计算新状态
        let new_state = self.get_current_state();
        
        // 5. 检查是否结束（这里简单设置为 false，实际可根据时间或其他条件）
        let done = false;
        
        (new_state, reward, done)
    }
    
    fn simulate_market_move(&mut self) {
        // 简单的几何布朗运动
        let normal = Normal::new(0.0, self.price_volatility).unwrap();
        let price_change = normal.sample(&mut rand::rng());
        
        let new_mid_ticks = ((self.last_mid_price + price_change) / self.tick_size) as i64;
        let old_mid_ticks = (self.last_mid_price / self.tick_size) as i64;
        let tick_change = new_mid_ticks - old_mid_ticks;
        
        if tick_change != 0 {
            self.shift_orderbook(tick_change);
            self.last_mid_price = new_mid_ticks as f64 * self.tick_size;
        }
    }
    
    fn shift_orderbook(&mut self, tick_change: i64) {
        // 简化处理：整体移动订单簿
        let old_bids: Vec<_> = self.orderbook.bids.clone().into_iter().collect();
        let old_asks: Vec<_> = self.orderbook.asks.clone().into_iter().collect();
        
        self.orderbook.bids.clear();
        self.orderbook.asks.clear();
        
        for (price, quantity) in old_bids {
            self.orderbook.bids.insert(price + tick_change, quantity);
        }
        
        for (price, quantity) in old_asks {
            self.orderbook.asks.insert(price + tick_change, quantity);
        }
    }
    
    fn execute_action(&mut self, action: Action) -> f64 {
        match action {
            Action::PlaceBid(distance) => {
                self.place_our_order(Side::Buy, distance);
            }
            Action::PlaceAsk(distance) => {
                self.place_our_order(Side::Sell, distance);
            }
            Action::PlaceBoth { bid_dist, ask_dist } => {
                self.place_our_order(Side::Buy, bid_dist);
                self.place_our_order(Side::Sell, ask_dist);
            }
            Action::CancelBids => {
                self.cancel_our_orders(Some(Side::Buy));
            }
            Action::CancelAsks => {
                self.cancel_our_orders(Some(Side::Sell));
            }
            Action::CancelAll => {
                self.cancel_our_orders(None);
            }
            Action::DoNothing => {}
        }
        
        0.0  // 动作本身不产生奖励，奖励来自后续的订单成交
    }
    
    fn place_our_order(&mut self, side: Side, distance: u8) {
        let mid_ticks = (self.last_mid_price / self.tick_size) as i64;
        let price = match side {
            Side::Buy => mid_ticks - distance as i64,
            Side::Sell => mid_ticks + distance as i64,
        };
        
        let order = Order {
            id: rand::random(),
            side,
            price,
            quantity: 100.0,
            timestamp: 0,  // 简化处理
        };
        
        self.orderbook.add_order(side, price, 100.0);
        self.our_orders.push(order);
    }
    
    fn cancel_our_orders(&mut self, side_filter: Option<Side>) {
        let (to_cancel, to_keep): (Vec<_>, Vec<_>) = self.our_orders
            .drain(..)
            .partition(|order| side_filter.map_or(true, |s| s == order.side));

        for order in &to_cancel {
            self.orderbook.remove_order(order.side, order.price, order.quantity);
        }

        self.our_orders = to_keep;
    }
    
    fn check_order_fills(&mut self) -> f64 {
        let mut total_reward = 0.0;

        // 先分离已成交和未成交的订单
        let (filled, remaining): (Vec<_>, Vec<_>) = self.our_orders
            .drain(..)
            .partition(|_| rand::rng().random::<f64>() < 0.1);

        self.our_orders = remaining;

        // 再处理成交逻辑
        for order in &filled {
            let trade_pnl = match order.side {
                Side::Buy => {
                    self.current_inventory += order.quantity as i32;
                    (self.last_mid_price - order.price as f64 * self.tick_size) * order.quantity
                }
                Side::Sell => {
                    self.current_inventory -= order.quantity as i32;
                    (order.price as f64 * self.tick_size - self.last_mid_price) * order.quantity
                }
            };

            let reward = self.reward_calculator.calculate_reward(
                trade_pnl,
                order.quantity as i32,
                self.current_inventory,
                false,
            );

            total_reward += reward;
        }

        total_reward
    }
    
    pub fn get_current_state(&self) -> MarketState {
        let spread = self.orderbook.spread().unwrap_or(2) as u8;
        let imbalance = (self.orderbook.imbalance() * 100.0) as i8;
        
        // 简化动量计算
        let momentum = 0i8;  
        
        let quote_state = match (
            self.our_orders.iter().any(|o| matches!(o.side, Side::Buy)),
            self.our_orders.iter().any(|o| matches!(o.side, Side::Sell)),
        ) {
            (true, true) => QuoteState::BothSides,
            (true, false) => QuoteState::BidOnly,
            (false, true) => QuoteState::AskOnly,
            (false, false) => QuoteState::NoQuotes,
        };
        
        MarketState {
            inventory: self.current_inventory.clamp(-10, 10) as i8,
            spread: spread.clamp(1, 20),
            imbalance: imbalance.clamp(-100, 100),
            momentum,
            quote_state,
        }
    }
}

训练循环

把所有组件整合起来进行训练：

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let mut agent = QLearningAgent::new(0.1, 0.95, 0.9);
    let mut env = MarketEnvironment::new(100.0, 0.01, 0.001);
    
    let episodes = 100000;
    let mut total_rewards = Vec::new();
    
    println!("开始训练做市智能体...");
    
    for episode in 0..episodes {
        let mut state = env.get_current_state();
        let mut episode_reward = 0.0;
        let max_steps = 1000;
        
        for _step in 0..max_steps {
            // 智能体选择动作
            let action = agent.choose_action(&state);
            
            // 执行动作，获取奖励和下一状态
            let (next_state, reward, done) = env.step(action);
            episode_reward += reward;
            
            // 更新 Q 值
            agent.update_q_value(&state, &action, reward, &next_state);
            
            state = next_state;
            
            if done {
                break;
            }
        }
        
        total_rewards.push(episode_reward);
        
        if episode % 1000 == 0 {
            let avg_reward: f64 = total_rewards.iter()
                .skip(total_rewards.len().saturating_sub(100))
                .sum::<f64>() / 100.0;
            
            println!("Episode {}: 平均奖励 = {:.2}, ε = {:.3}", 
                     episode, avg_reward, agent.epsilon);
        }
    }
    
    // 保存训练好的 Q 表
    save_q_table(&agent.q_table, "market_making_q_table.json")?;
    
    println!("训练完成！");
    Ok(())
}

fn save_q_table(
    q_table: &HashMap<(MarketState, Action), f64>,
    filename: &str,
) -> Result<(), Box<dyn std::error::Error>> {
    use std::fs::File;
    use std::io::Write;
    
    let mut file = File::create(filename)?;
    
    // 简化序列化：只保存非零的 Q 值
    for ((state, action), &q_value) in q_table.iter() {
        if q_value.abs() > 1e-6 {
            writeln!(file, "{:?},{:?},{:.6}", state, action, q_value)?;
        }
    }
    
    Ok(())
}

2. 风险管理

pub struct RiskManager {
    max_inventory: i32,
    max_daily_loss: f64,
    current_daily_pnl: f64,
    position_limit_per_trade: f64,
}

impl RiskManager {
    pub fn check_action_allowed(
        &self,
        action: &Action,
        current_inventory: i32,
        estimated_impact: f64,
    ) -> bool {
        // 1. 库存限制检查
        match action {
            Action::PlaceBid(_) => {
                if current_inventory >= self.max_inventory {
                    return false;
                }
            }
            Action::PlaceAsk(_) => {
                if current_inventory <= -self.max_inventory {
                    return false;
                }
            }
            _ => {}
        }
        
        // 2. 日损失限制
        if self.current_daily_pnl <= -self.max_daily_loss {
            return false;
        }
        
        // 3. 单笔交易限制
        if estimated_impact.abs() > self.position_limit_per_trade {
            return false;
        }
        
        true
    }
}

use std::time::{Duration, Instant};

pub struct PerformanceMonitor {
    trade_count: u64,
    total_pnl: f64,
    max_drawdown: f64,
    sharpe_ratio: f64,
    last_update: Instant,
}

impl PerformanceMonitor {
    pub fn record_trade(&mut self, pnl: f64) {
        self.trade_count += 1;
        self.total_pnl += pnl;
        
        // 更新最大回撤
        self.max_drawdown = self.max_drawdown.min(self.total_pnl);
        
        // 定期计算夏普比率
        if self.last_update.elapsed() > Duration::from_secs(3600) {
            self.calculate_sharpe_ratio();
            self.last_update = Instant::now();
        }
    }
    
    fn calculate_sharpe_ratio(&mut self) {
        // 简化计算，实际应该用小时级别的收益率
        if self.trade_count > 100 {
            let avg_return = self.total_pnl / self.trade_count as f64;
            // 这里需要计算收益率的标准差，简化处理
            self.sharpe_ratio = avg_return / 0.01;  // 假设波动率 1%
        }
    }
}

性能测试结果

在我的测试环境中（Mac Studio，Apple M2 Max，32GB RAM，macOS 15.7.3，rustc 1.94.0），整个训练过程的性能表现：

  • 状态转换速度：约 10–20 微秒/步（中位数约 15 微秒）
  • Q 表更新：平均 8 微秒/次
  • 订单簿处理：平均 12 微秒/更新
  • 内存使用：Q 表约占 200MB（100K episodes 后）

相比我之前用 Python 写的版本，速度提升了大约 20-30 倍，内存使用也减少了不少。

下一步改进

这个实现还比较基础，后续可以考虑：

  1. 更复杂的状态表示：加入市场微观结构特征（大单预警、交易量模式等）
  2. 深度 Q 网络：用神经网络替代 Q 表，处理连续状态空间
  3. 多智能体训练：模拟多个做市商之间的博弈
  4. 更真实的市场模拟：基于历史 tick 数据进行回测
  5. 在线学习：实盘部署后继续学习和调整策略

强化学习在做市策略上确实有潜力，但真正用到生产环境还需要大量的测试和优化。不过用 Rust 来实现至少在性能上不会成为瓶颈，这为后续的复杂策略开发打下了好基础。
