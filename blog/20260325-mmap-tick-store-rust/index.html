用 mmap 存 Tick 数据：一个 32MB 文件撑住百万级行情回放

⏱ 2026年3月25日
· 39 分钟阅读
#Rust#mmap#行情数据#交易系统#性能优化

最近在做回测框架的时候遇到一个老问题：tick 数据怎么存。

情况是这样的——一个交易对一天大概 200 万到 500 万条 tick，我需要按时间范围快速切片读取。之前的方案是 SQLite，每次回测一个策略要花 40 多秒在数据加载上，实际策略执行才几秒。瓶颈全在 IO。

试过 Parquet，读取速度确实快很多，但我的场景是纯追加写入 + 按时间范围读取，Parquet 的列式存储优势发挥不出来——我每条 tick 的所有字段都要用。而且 Parquet 的 Rust 生态（parquet crate）依赖链太重了，编译一次要等半天。

后来想到最简单的办法：直接把 tick 结构体按固定布局写进文件，用 mmap 映射进来。读取就是指针偏移，没有任何解析开销。

数据布局设计

先把最关键的问题想清楚：一条 tick 需要哪些字段，占多少字节。

use bytemuck::{Pod, Zeroable};

/// 文件头，固定 64 字节
#[derive(Debug, Clone, Copy, Pod, Zeroable)]
#[repr(C)]
struct FileHeader {
    magic: [u8; 4],       // b"TICK"
    version: u32,         // 1
    tick_count: u64,      // 当前 tick 数量
    start_ts: u64,        // 第一条 tick 的时间戳
    end_ts: u64,          // 最后一条 tick 的时间戳
    _reserved: [u8; 32],  // 留给未来扩展
}

/// 单条 Tick 数据，32 字节
#[derive(Debug, Clone, Copy, Pod, Zeroable)]
#[repr(C)]
struct Tick {
    timestamp_us: u64,   // 微秒时间戳
    price: u64,          // 价格 × 10^8（定点数）
    quantity: u64,       // 数量 × 10^8
    flags: u32,          // bit0: is_buy, 其余保留
    _padding: u32,       // 对齐到 32 字节
}

几个设计决策：

定点数而不是浮点数。 价格用 u64 存储，乘以 10^8。加密货币价格精度到 8 位小数足够了，避免浮点比较的各种坑。42000.12345678 存成 4200012345678u64。

32 字节对齐。 一条 tick 刚好 32 字节，CPU cache line 通常 64 字节，一个 cache line 装两条 tick。这不是巧合，是故意凑的。

_reserved 字段。 64 字节的 header 现在只用了 32 字节，剩下的留着。以后想加字段（比如交易对名称、压缩标志）直接往 reserved 里填，老文件还能读。

#[repr(C)] + Pod。 repr(C) 保证内存布局确定，Pod（bytemuck 提供）保证类型可以安全地和 &[u8] 互转。这两个加一起，就能用 mmap 直接把文件映射成结构体切片，零拷贝。

编译期检查一下大小没算错：

const HEADER_SIZE: usize = std::mem::size_of::<FileHeader>();
const TICK_SIZE: usize = std::mem::size_of::<Tick>();

const _: () = assert!(HEADER_SIZE == 64);
const _: () = assert!(TICK_SIZE == 32);

TickStore 实现

核心就是一个 MmapMut，指向一个按上面布局组织的文件。

use memmap2::{MmapMut, MmapOptions};
use std::fs::{File, OpenOptions};
use std::io;
use std::path::Path;

struct TickStore {
    mmap: MmapMut,
    file: File,
    capacity: usize,
}

创建和打开

impl TickStore {
    fn create(path: &Path, initial_capacity: usize) -> io::Result<Self> {
        let file_size = HEADER_SIZE + initial_capacity * TICK_SIZE;
        let file = OpenOptions::new()
            .read(true)
            .write(true)
            .create(true)
            .truncate(true)
            .open(path)?;
        file.set_len(file_size as u64)?;

        let mut mmap = unsafe { MmapOptions::new().map_mut(&file)? };

        let header = FileHeader {
            magic: *b"TICK",
            version: 1,
            tick_count: 0,
            start_ts: 0,
            end_ts: 0,
            _reserved: [0u8; 32],
        };
        mmap[..HEADER_SIZE].copy_from_slice(bytemuck::bytes_of(&header));
        mmap.flush()?;

        Ok(Self {
            mmap,
            file,
            capacity: initial_capacity,
        })
    }

    fn open(path: &Path) -> io::Result<Self> {
        let file = OpenOptions::new().read(true).write(true).open(path)?;
        let mmap = unsafe { MmapOptions::new().map_mut(&file)? };

        let header: &FileHeader = bytemuck::from_bytes(&mmap[..HEADER_SIZE]);
        if &header.magic != b"TICK" {
            return Err(io::Error::new(
                io::ErrorKind::InvalidData,
                "bad magic",
            ));
        }

        let capacity = (mmap.len() - HEADER_SIZE) / TICK_SIZE;
        Ok(Self { mmap, file, capacity })
    }
}

unsafe 在 mmap 这里绑不开。memmap2 的文档也说得很清楚——如果另一个进程同时修改文件，你读到的数据就是 UB。在我的场景里，写入是单线程追加，回测时只读不写，所以没问题。

追加写入

impl TickStore {
    fn push(&mut self, tick: Tick) -> io::Result<()> {
        let count = self.tick_count();
        if count >= self.capacity {
            self.grow()?;
        }

        let offset = HEADER_SIZE + count * TICK_SIZE;
        self.mmap[offset..offset + TICK_SIZE]
            .copy_from_slice(bytemuck::bytes_of(&tick));

        let header = self.header_mut();
        header.tick_count = (count + 1) as u64;
        if count == 0 {
            header.start_ts = tick.timestamp_us;
        }
        header.end_ts = tick.timestamp_us;

        Ok(())
    }

    fn grow(&mut self) -> io::Result<()> {
        let new_capacity = self.capacity * 2;
        let new_size = HEADER_SIZE + new_capacity * TICK_SIZE;
        self.file.set_len(new_size as u64)?;
        self.mmap = unsafe { MmapOptions::new().map_mut(&self.file)? };
        self.capacity = new_capacity;
        Ok(())
    }
}

扩容策略直接翻倍，和 Vec 一样。初始容量给 1024 的话，写到 100 万条只需要扩容 10 次。每次扩容要重新 mmap，但这个操作本身很快——只是改页表映射，不拷贝数据。

O(1) 随机读取

impl TickStore {
    fn get(&self, index: usize) -> Option<&Tick> {
        if index >= self.tick_count() {
            return None;
        }
        let offset = HEADER_SIZE + index * TICK_SIZE;
        Some(bytemuck::from_bytes(
            &self.mmap[offset..offset + TICK_SIZE],
        ))
    }
}

这是 mmap 方案最爽的地方。读第 50 万条 tick 和读第 1 条，代码路径完全一样，就是一个偏移量计算。没有 seek，没有 read syscall，直接访问内存地址，OS 的 page cache 搞定一切。

按时间二分查找

tick 数据天然按时间递增排列，直接二分：

impl TickStore {
    fn find_by_timestamp(&self, target_us: u64) -> Option<usize> {
        let count = self.tick_count();
        if count == 0 {
            return None;
        }
        let mut lo = 0usize;
        let mut hi = count;
        while lo < hi {
            let mid = lo + (hi - lo) / 2;
            let tick = self.get(mid).unwrap();
            if tick.timestamp_us < target_us {
                lo = mid + 1;
            } else {
                hi = mid;
            }
        }
        if lo < count { Some(lo) } else { None }
    }
}

在 100 万条数据里找一个时间戳，最多比较 20 次（log₂(1000000) ≈ 20），每次比较就是一个内存访问。

实际跑一下

测试环境：

  • Mac Studio, Apple M2 Max, 32 GB
  • macOS 26.3.1
  • rustc 1.94.0 (4a4ef493e 2026-03-02)

写入 100 万条 tick 到文件，然后测随机读取和二分查找：

fn main() -> io::Result<()> {
    let path = Path::new("/tmp/ticks_demo.bin");
    let n = 1_000_000usize;

    // 写入
    let start = Instant::now();
    {
        let mut store = TickStore::create(path, 1024)?;
        for i in 0..n {
            let tick = make_tick(
                1_700_000_000_000_000 + i as u64 * 100,
                42000.0 + (i as f64 * 0.01).sin() * 50.0,
                0.1 + (i % 100) as f64 * 0.001,
                i % 3 != 0,
            );
            store.push(tick)?;
        }
        store.flush()?;
    }
    println!("写入: {:.2}ms", start.elapsed().as_secs_f64() * 1000.0);

    // 随机读取
    let store = TickStore::open(path)?;
    let start = Instant::now();
    let mut sum_price = 0u64;
    for i in (0..n).step_by(7) {
        sum_price += store.get(i).unwrap().price;
    }
    println!(
        "随机读 {} 条: {:.2}ms",
        n / 7,
        start.elapsed().as_secs_f64() * 1000.0
    );

    // 二分查找
    let target = 1_700_000_000_000_000 + 500_000 * 100;
    let start = Instant::now();
    let idx = store.find_by_timestamp(target);
    println!("二分查找: {:.2}μs", start.elapsed().as_secs_f64() * 1e6);

    Ok(())
}

运行结果（release 编译）：

写入 1000000 条 tick: 23.07ms, 文件大小: 32.00 MB
随机读取 142857 条: 1.53ms
二分查找: 0.96μs

100 万条 tick 写入 23ms，文件 32MB。随机读 14 万条 1.5ms。二分查找不到 1 微秒。

32MB 放 100 万条——一天的 tick 数据（假设 300 万条）也就 96MB，一个月不到 3GB。对比 SQLite 一天的数据轻松上 500MB，这个体积小太多了。

时间片读取

回测时最常见的操作是”取某个时间范围内的所有 tick”。有了二分查找，这就是找到起点然后顺序遍历：

let from_ts = 1_700_000_000_000_000 + 100_000 * 100;
let to_ts = 1_700_000_000_000_000 + 100_100 * 100;
let start_idx = store.find_by_timestamp(from_ts).unwrap_or(0);
let mut slice_ticks = Vec::new();
for i in start_idx.. {
    match store.get(i) {
        Some(t) if t.timestamp_us <= to_ts => slice_ticks.push(t),
        _ => break,
    }
}

因为数据在文件里是连续存放的，顺序读取对 page cache 非常友好。OS 会预读后续的页面，实际的磁盘 IO 很少。

踩过的坑

bytemuck 版本。 bytemuck 1.x 里 Pod derive 要求所有字段都实现 Pod。[u8; 32] 没问题，但如果你不小心用了 bool 就会编译失败——bool 不是 Pod，因为它在 C ABI 里的合法值只有 0 和 1，但 mmap 读出来的字节可以是任意值。所以我用 u32 的 bit flag 代替布尔字段。

扩容时的 mmap 失效。 grow() 里先 set_len 扩大文件，再重新 mmap。这里有个细节：旧的 MmapMut 在被替换时会自动 unmap（Drop），但如果你之前持有 get() 返回的引用，那个引用就悬空了。Rust 的借用检查器帮你挡住了这个问题——push 拿了 &mut self，get 返回 &Tick 绑定到 &self，两者不可能同时存在。这也是为什么我把 mmap 和 file 放在同一个结构体里，让 borrow checker 自然保护。

_reserved 字段的对齐。 一开始我只留了 28 字节 reserved，header 总共 60 字节。运行没问题，但后来意识到如果 header 不按 cache line 对齐，第一条 tick 数据会跨 cache line。多加了 4 字节凑到 64。

什么时候不该用这个方案

说几个这套方案不合适的场景：

  • 需要删除或修改历史数据。 定长记录的文件格式不支持中间删除，只能追加。如果要删，得重写整个文件。
  • 跨平台共享数据文件。 repr(C) 保证了结构体布局，但字节序取决于平台。M2 是小端，如果你要把数据文件拷贝到大端机器上就得转换。不过 2026 年了，谁还在用大端呢。
  • 多进程并发写入。 mmap 在多进程写入时没有内建的锁机制，你得自己加文件锁或者用 advisory lock。我的场景是单写多读，不需要操心这个。

完整代码

use bytemuck::{Pod, Zeroable};
use memmap2::{MmapMut, MmapOptions};
use std::fs::{File, OpenOptions};
use std::io;
use std::mem;
use std::path::Path;
use std::time::Instant;

#[derive(Debug, Clone, Copy, Pod, Zeroable)]
#[repr(C)]
struct FileHeader {
    magic: [u8; 4],
    version: u32,
    tick_count: u64,
    start_ts: u64,
    end_ts: u64,
    _reserved: [u8; 32],
}

#[derive(Debug, Clone, Copy, Pod, Zeroable)]
#[repr(C)]
struct Tick {
    timestamp_us: u64,
    price: u64,
    quantity: u64,
    flags: u32,
    _padding: u32,
}

const HEADER_SIZE: usize = mem::size_of::<FileHeader>();
const TICK_SIZE: usize = mem::size_of::<Tick>();
const _: () = assert!(HEADER_SIZE == 64);
const _: () = assert!(TICK_SIZE == 32);

struct TickStore {
    mmap: MmapMut,
    file: File,
    capacity: usize,
}

impl TickStore {
    fn create(path: &Path, initial_capacity: usize) -> io::Result<Self> {
        let file_size = HEADER_SIZE + initial_capacity * TICK_SIZE;
        let file = OpenOptions::new()
            .read(true)
            .write(true)
            .create(true)
            .truncate(true)
            .open(path)?;
        file.set_len(file_size as u64)?;

        let mut mmap = unsafe { MmapOptions::new().map_mut(&file)? };

        let header = FileHeader {
            magic: *b"TICK",
            version: 1,
            tick_count: 0,
            start_ts: 0,
            end_ts: 0,
            _reserved: [0u8; 32],
        };
        mmap[..HEADER_SIZE].copy_from_slice(bytemuck::bytes_of(&header));
        mmap.flush()?;

        Ok(Self { mmap, file, capacity: initial_capacity })
    }

    fn open(path: &Path) -> io::Result<Self> {
        let file = OpenOptions::new().read(true).write(true).open(path)?;
        let mmap = unsafe { MmapOptions::new().map_mut(&file)? };

        let header: &FileHeader =
            bytemuck::from_bytes(&mmap[..HEADER_SIZE]);
        if &header.magic != b"TICK" {
            return Err(io::Error::new(
                io::ErrorKind::InvalidData,
                "bad magic",
            ));
        }

        let capacity = (mmap.len() - HEADER_SIZE) / TICK_SIZE;
        Ok(Self { mmap, file, capacity })
    }

    fn header(&self) -> &FileHeader {
        bytemuck::from_bytes(&self.mmap[..HEADER_SIZE])
    }

    fn header_mut(&mut self) -> &mut FileHeader {
        bytemuck::from_bytes_mut(&mut self.mmap[..HEADER_SIZE])
    }

    fn tick_count(&self) -> usize {
        self.header().tick_count as usize
    }

    fn push(&mut self, tick: Tick) -> io::Result<()> {
        let count = self.tick_count();
        if count >= self.capacity {
            self.grow()?;
        }

        let offset = HEADER_SIZE + count * TICK_SIZE;
        self.mmap[offset..offset + TICK_SIZE]
            .copy_from_slice(bytemuck::bytes_of(&tick));

        let header = self.header_mut();
        header.tick_count = (count + 1) as u64;
        if count == 0 {
            header.start_ts = tick.timestamp_us;
        }
        header.end_ts = tick.timestamp_us;

        Ok(())
    }

    fn get(&self, index: usize) -> Option<&Tick> {
        if index >= self.tick_count() {
            return None;
        }
        let offset = HEADER_SIZE + index * TICK_SIZE;
        Some(bytemuck::from_bytes(
            &self.mmap[offset..offset + TICK_SIZE],
        ))
    }

    fn find_by_timestamp(&self, target_us: u64) -> Option<usize> {
        let count = self.tick_count();
        if count == 0 {
            return None;
        }
        let mut lo = 0usize;
        let mut hi = count;
        while lo < hi {
            let mid = lo + (hi - lo) / 2;
            let tick = self.get(mid).unwrap();
            if tick.timestamp_us < target_us {
                lo = mid + 1;
            } else {
                hi = mid;
            }
        }
        if lo < count { Some(lo) } else { None }
    }

    fn grow(&mut self) -> io::Result<()> {
        let new_capacity = self.capacity * 2;
        let new_size = HEADER_SIZE + new_capacity * TICK_SIZE;
        self.file.set_len(new_size as u64)?;
        self.mmap = unsafe { MmapOptions::new().map_mut(&self.file)? };
        self.capacity = new_capacity;
        Ok(())
    }

    fn flush(&self) -> io::Result<()> {
        self.mmap.flush()
    }
}

fn make_tick(ts_us: u64, price_f64: f64, qty_f64: f64, is_buy: bool) -> Tick {
    Tick {
        timestamp_us: ts_us,
        price: (price_f64 * 1e8) as u64,
        quantity: (qty_f64 * 1e8) as u64,
        flags: if is_buy { 1 } else { 0 },
        _padding: 0,
    }
}

fn price_f64(tick: &Tick) -> f64 {
    tick.price as f64 / 1e8
}

fn main() -> io::Result<()> {
    let path = Path::new("/tmp/ticks_demo.bin");
    let n = 1_000_000usize;

    println!("写入 {n} 条 tick 到 {}", path.display());
    let start = Instant::now();
    {
        let mut store = TickStore::create(path, 1024)?;
        for i in 0..n {
            let tick = make_tick(
                1_700_000_000_000_000 + i as u64 * 100,
                42000.0 + (i as f64 * 0.01).sin() * 50.0,
                0.1 + (i % 100) as f64 * 0.001,
                i % 3 != 0,
            );
            store.push(tick)?;
        }
        store.flush()?;
    }
    let write_elapsed = start.elapsed();
    let file_size = std::fs::metadata(path)?.len();
    println!(
        "写入完成: {:.2}ms, 文件大小: {:.2} MB",
        write_elapsed.as_secs_f64() * 1000.0,
        file_size as f64 / 1024.0 / 1024.0
    );

    let store = TickStore::open(path)?;
    println!("tick 总数: {}", store.tick_count());

    let start = Instant::now();
    let mut sum_price = 0u64;
    for i in (0..n).step_by(7) {
        sum_price += store.get(i).unwrap().price;
    }
    let read_elapsed = start.elapsed();
    println!(
        "随机读取 {} 条: {:.2}ms (防优化: {})",
        n / 7,
        read_elapsed.as_secs_f64() * 1000.0,
        sum_price
    );

    let target = 1_700_000_000_000_000 + 500_000 * 100;
    let start = Instant::now();
    let idx = store.find_by_timestamp(target);
    let search_elapsed = start.elapsed();
    if let Some(i) = idx {
        let tick = store.get(i).unwrap();
        println!(
            "二分查找 ts={target}: index={i}, price={:.2}, 耗时: {:.2}μs",
            price_f64(tick),
            search_elapsed.as_secs_f64() * 1e6
        );
    }

    std::fs::remove_file(path)?;
    println!("done.");
    Ok(())
}
