从零用 Rust 实现 Black-Scholes 期权定价引擎

⏱ 2026年3月15日
· 27 分钟阅读
#rust#quantitative-finance#options#monte-carlo

期权定价是量化金融的基础课，但很少有人真正从头写过一遍。用 QuantLib、scipy.stats.norm 或者 py_vollib 调个函数，看到数字出来就觉得懂了——其实未必。

这篇文章用纯 Rust 实现 Black-Scholes 定价引擎，不用任何外部 crate，连正态分布都自己算。然后写 Monte Carlo 做数值验证，最后用 Put-Call Parity 做恒等式校验。全程代码可运行，测试覆盖核心边界条件。

环境

rustc 1.94.0 (4a4ef493e 2026-03-02)
cargo 1.94.0 (85eff7c80 2026-01-15)
macOS 15.7.3 (arm64)

Black-Scholes 公式回顾

对于欧式期权，Black-Scholes 公式给出解析解：

Call: $$C = S e^{-qT} N(d_1) - K e^{-rT} N(d_2)$$

Put: $$P = K e^{-rT} N(-d_2) - S e^{-qT} N(-d_1)$$

其中： $$d_1 = \frac{\ln(S/K) + (r - q + \sigma^2/2)T}{\sigma\sqrt{T}}, \quad d_2 = d_1 - \sigma\sqrt{T}$$

  • $S$：标的资产现价
  • $K$：行权价
  • $r$：无风险利率（连续复利）
  • $q$：连续股息率
  • $\sigma$：波动率
  • $T$：到期时间（年）
  • $N(\cdot)$：标准正态分布 CDF

公式本身不复杂，但实现时有几个细节容易踩坑：正态 CDF 的精度、Greeks 的 Theta 要不要按日折算、有股息时 delta 的表达式。下面一一处理。

正态 CDF 不依赖外部库怎么写

这是第一个问题。Python 有 scipy.stats.norm.cdf，Rust 标准库没有。

常见做法是用误差函数 erfc（libm 提供），但 libm crate 也是外部依赖。其实 Abramowitz & Stegun 的 26.2.17 给了一个多项式近似，最大误差 $7.5 \times 10^{-8}$，对期权定价完全够用：

use std::f64::consts::PI;

fn norm_cdf(x: f64) -> f64 {
    let t = 1.0 / (1.0 + 0.2316419 * x.abs());
    let poly = t * (0.319381530
        + t * (-0.356563782
        + t * (1.781477937
        + t * (-1.821255978
        + t * 1.330274429))));
    let cdf = 1.0 - ((-0.5 * x * x).exp() / (2.0 * PI).sqrt()) * poly;
    if x >= 0.0 { cdf } else { 1.0 - cdf }
}

fn norm_pdf(x: f64) -> f64 {
    (-0.5 * x * x).exp() / (2.0 * PI).sqrt()
}

这个近似用 Horner 方法展开，5 个系数，精度足够。实盘代码里我测了一下和 statrs crate 的结果，差距在 8 位小数之后，期权价格误差约 $10^{-9}$ 量级，完全可以接受。

值得一说的是 Horner 形式：

t * (a1 + t * (a2 + t * (a3 + t * (a4 + t * a5))))

等价于多项式求值，但只需要 4 次乘法而非朴素展开的 15 次。这在批量定价（比如同时算几千条 strike）时能省不少时间，虽然现代编译器会优化大部分情况，但手写 Horner 是个好习惯。

还有一个细节：对于负数 x，直接套公式会出问题——Abramowitz & Stegun 的近似只对 $x \geq 0$ 成立。所以代码里用 x.abs()，再用 1 - CDF(|x|) 对称处理负数侧。这个分支是必须的，不能省。

数据结构

#[derive(Debug, Clone, Copy, PartialEq)]
pub enum OptionType {
    Call,
    Put,
}

#[derive(Debug, Clone)]
pub struct BSParams {
    pub s: f64,     // 标的资产现价
    pub k: f64,     // 行权价
    pub r: f64,     // 无风险利率（年化）
    pub sigma: f64, // 隐含波动率（年化）
    pub t: f64,     // 到期时间（年）
    pub q: f64,     // 连续股息率
}

#[derive(Debug, Clone)]
pub struct Greeks {
    pub price: f64,
    pub delta: f64,
    pub gamma: f64,
    pub theta: f64, // 每日 theta
    pub vega: f64,  // 每 1% 波动率变化的 vega
    pub rho: f64,   // 每 1% 利率变化的 rho
}

BSParams 里加了股息率 q。不带股息的公式令 q = 0 即可退化。

定价与 Greeks

impl BSParams {
    fn d1(&self) -> f64 {
        ((self.s / self.k).ln() + (self.r - self.q + 0.5 * self.sigma * self.sigma) * self.t)
            / (self.sigma * self.t.sqrt())
    }

    fn d2(&self) -> f64 {
        self.d1() - self.sigma * self.t.sqrt()
    }

    pub fn greeks(&self, opt: OptionType) -> Greeks {
        let d1 = self.d1();
        let d2 = self.d2();
        let df  = (-self.r * self.t).exp();  // 折现因子
        let qf  = (-self.q * self.t).exp();  // 股息折现

        let nd1     = norm_cdf(d1);
        let nd2     = norm_cdf(d2);
        let nd1_neg = norm_cdf(-d1);
        let nd2_neg = norm_cdf(-d2);
        let pdf_d1  = norm_pdf(d1);

        let (price, delta, rho) = match opt {
            OptionType::Call => (
                self.s * qf * nd1 - self.k * df * nd2,
                qf * nd1,
                self.k * self.t * df * nd2 / 100.0,
            ),
            OptionType::Put => (
                self.k * df * nd2_neg - self.s * qf * nd1_neg,
                -qf * nd1_neg,
                -self.k * self.t * df * nd2_neg / 100.0,
            ),
        };

        let gamma = qf * pdf_d1 / (self.s * self.sigma * self.t.sqrt());

        // Theta 除以 365 转换为每日衰减
        let theta = match opt {
            OptionType::Call => {
                (-self.s * qf * pdf_d1 * self.sigma / (2.0 * self.t.sqrt())
                    - self.r * self.k * df * nd2
                    + self.q * self.s * qf * nd1) / 365.0
            }
            OptionType::Put => {
                (-self.s * qf * pdf_d1 * self.sigma / (2.0 * self.t.sqrt())
                    + self.r * self.k * df * nd2_neg
                    - self.q * self.s * qf * nd1_neg) / 365.0
            }
        };
        let vega = self.s * qf * pdf_d1 * self.t.sqrt() / 100.0;

        Greeks { price, delta, gamma, theta, vega, rho }
    }
}

几个容易弄错的地方：

Gamma 对 call 和 put 相同。数学上两者的二阶导数一致：$\Gamma = e^{-qT} N’(d_1) / (S\sigma\sqrt{T})$，与期权类型无关。所以代码里 gamma 的计算放在 match 外面，call/put 共用同一行。

Theta 的符号约定。有些教材用正数表示”时间价值每天减少多少”，有些用负数表示”持有一天组合价值变化”。这里用后者，所以 call theta 通常是负数（每天损失时间价值）。

Vega 和 Rho 的单位。原始公式给出的是对 sigma 的一阶导，即”波动率变化 1 个单位（100%）时的价格变化”。除以 100 转换成”每 1% 波动率变化的 vega”，这是市场惯例。Rho 同理。

实际含义快速对照

看 ATM call 的输出结果来理解这些数字：

Delta: 0.6368  ← 标的涨 1 元，call 涨 0.64 元（ATM call 约 0.5，这里略高因为 r>0）
Gamma: 0.0188  ← 标的每涨 1 元，delta 本身增加 0.019
Theta: -0.0176 ← 每过一天，call 损失约 1.76 分（时间价值衰减）
Vega:  0.3752  ← 波动率每升 1%，call 涨 0.375 元
Rho:   0.5323  ← 利率每升 1%，call 涨 0.532 元

做期权做市商要管的是整个组合的 Greeks exposure 而不是单条腿。比如把 delta 对冲掉后，剩下 gamma/vega 的 P&L 才是主要来源。Theta 是代价——你 long gamma 就要付 theta，这个 trade-off 是期权交易的核心张力。

用 Monte Carlo 验证

解析解有了，怎么验证对不对？Monte Carlo 是最直观的方法：在风险中性测度下模拟资产价格路径，算平均折现收益。

两种方法是独立的，如果都收敛到同一个数，基本说明公式没算错。

pub fn monte_carlo_call(params: &BSParams, n: usize, seed: u64) -> f64 {
    let df    = (-params.r * params.t).exp();
    let drift = (params.r - params.q - 0.5 * params.sigma * params.sigma) * params.t;
    let vol   = params.sigma * params.t.sqrt();

    let mut state = seed;
    let mut sum = 0.0f64;

    for _ in 0..n {
        // Box-Muller 变换生成标准正态随机数
        state = state.wrapping_mul(6364136223846793005).wrapping_add(1442695040888963407);
        let u1 = (state >> 33) as f64 / (u64::MAX >> 33) as f64;
        state = state.wrapping_mul(6364136223846793005).wrapping_add(1442695040888963407);
        let u2 = (state >> 33) as f64 / (u64::MAX >> 33) as f64;
        let z = (-2.0 * u1.ln()).sqrt() * (2.0 * PI * u2).cos();

        let st = params.s * (drift + vol * z).exp();
        sum += (st - params.k).max(0.0);
    }

    df * sum / n as f64
}

随机数用 LCG（线性同余生成器）加 Box-Muller 变换。不是最好的 RNG，但用于验证足够，而且可复现（固定 seed）。

测试

#[cfg(test)]
mod tests {
    use super::*;

    fn default_params() -> BSParams {
        BSParams { s: 100.0, k: 100.0, r: 0.05, sigma: 0.2, t: 1.0, q: 0.0 }
    }

    #[test]
    fn test_call_price_hull_example() {
        let p = BSParams { s: 42.0, k: 40.0, r: 0.10, sigma: 0.20, t: 0.5, q: 0.0 };
        let call = p.greeks(OptionType::Call);
        assert!((call.price - 4.76).abs() < 0.01, "call price: {}", call.price);
    }

    #[test]
    fn test_put_call_parity() {
        let p = default_params();
        let call = p.greeks(OptionType::Call);
        let put  = p.greeks(OptionType::Put);
        let lhs = call.price - put.price;
        let rhs = p.s - p.k * (-p.r * p.t).exp();
        assert!((lhs - rhs).abs() < 1e-10);
    }

    #[test]
    fn test_gamma_positive() {
        let p = default_params();
        let call = p.greeks(OptionType::Call);
        let put  = p.greeks(OptionType::Put);
        assert!(call.gamma > 0.0);
        assert!((call.gamma - put.gamma).abs() < 1e-10);
    }

    #[test]
    fn test_deep_itm_call_delta() {
        let p = BSParams { s: 200.0, k: 100.0, r: 0.05, sigma: 0.2, t: 1.0, q: 0.0 };
        let call = p.greeks(OptionType::Call);
        assert!(call.delta > 0.95);
    }

    #[test]
    fn test_expired_call_intrinsic_value() {
        // T 趋近 0，call ≈ max(S-K, 0)
        let p = BSParams { s: 110.0, k: 100.0, r: 0.05, sigma: 0.2, t: 0.001, q: 0.0 };
        let call = p.greeks(OptionType::Call);
        assert!((call.price - 10.0).abs() < 0.05);
    }

    #[test]
    fn test_monte_carlo_convergence() {
        let p = default_params();
        let bs = p.greeks(OptionType::Call).price;
        let mc = monte_carlo_call(&p, 2_000_000, 12345);
        assert!((mc - bs).abs() < 0.1);
    }
}

接下来可以做什么

这个实现只覆盖了欧式期权的 BS 定价。几个延伸方向：

隐含波动率反求（IV）。给定市场价格反推 σ，需要数值求解。Newton-Raphson 很适合，用 Vega 做除法：

pub fn implied_vol_call(params: &BSParams, market_price: f64) -> Option<f64> {
    let mut sigma = 0.2; // 初始猜测
    for _ in 0..100 {
        let p = BSParams { sigma, ..params.clone() };
        let g = p.greeks(OptionType::Call);
        let diff = g.price - market_price;
        if diff.abs() < 1e-8 { return Some(sigma); }
        // vega 是每 1% 的变化，反求时需要乘回 100
        let vega_raw = g.vega * 100.0;
        if vega_raw.abs() < 1e-10 { break; } // vega 接近 0，放弃
        sigma -= diff / vega_raw;
        if sigma <= 0.0 { sigma = 1e-6; } // 避免负波动率
    }
    None // 不收敛
}

这里 vega_raw 要乘回 100，因为 Greeks.vega 已经除以 100 换成”每 1% 单位”，Newton 迭代需要的是原始导数。这个细节如果没注意到，IV 就会收敛到错误的值——我第一版就踩了这个坑，迭代 100 次还是发散。

美式期权。BS 公式对美式期权不直接适用，需要 binomial tree 或 finite difference。Rust 写 binomial tree 很直接，性能也好。

波动率曲面。现实中 IV 随行权价和到期日变化（smile/skew），要做好期权定价需要 stochastic vol 模型（Heston、SABR 等），那就是另一篇的内容了。

为什么 BS 在实际中还在用

BS 的假设很强：对数正态分布、恒定波动率、连续对冲。这些在现实中全部不成立。

但 BS 作为定价语言仍然有用。市场参与者用 IV 报价，本质是在用 BS 公式把价格”翻译”成一个更直觉的数字。当交易员说”这个 call 的 vol 是 25”，意思就是用 25% 的 BS IV 能算出这个市价。

这也解释了为什么 IV 的计算永远不会过时：就算你用 Heston 或者机器学习模型定价，最终报出去的仍然是 BS IV，因为对手盘也用同一套语言。

所以理解 BS 的核心不是相信它的假设，而是知道它的 IV 是一个市场通用的 risk proxy。学好 BS greeks，才能理解更复杂模型的输出在说什么。
